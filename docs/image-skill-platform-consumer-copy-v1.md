# 图像 Skill 平台 V1 · 普通用户版文案与按钮文案清单

## 文档说明

- 文档目标：统一普通用户版首页、创作页、结果页的标题文案、说明文案、按钮文案、空态文案与失败态文案。
- 文档定位：承接 [image-skill-platform-consumer-interactions-v1.md](./image-skill-platform-consumer-interactions-v1.md) 的原型级文案文档。
- 当前状态：普通用户版文案规范文档。
- 更新时间：2026-05-05。

## 1. 文档原则

普通用户版文案必须遵循以下原则：

- 说人话，不说系统话
- 说动作，不说能力名词
- 说结果，不说技术过程
- 让用户轻松开始，而不是增加心理负担
- 失败时给出可执行下一步，而不是只给错误提示

一句话原则：

让用户感觉“有人在帮我做图”，而不是“我在操作一套复杂系统”。

## 2. 全局文案规范

### 2.1 优先使用的表达方式

- 从文字生成图片
- 按现有图片继续修改
- 只改这一部分
- 换成新背景
- 帮我补全细节
- 先试试看
- 继续做
- 再生成几张

### 2.2 避免直接暴露的表达

- Skill
- Prompt
- Rendered Prompt
- Provider
- Runtime Mode
- 模板 DSL
- 参数编排
- 图像工作流模板

### 2.3 语气要求

- 简短
- 明确
- 友好但不幼稚
- 不夸张承诺

### 2.4 按钮命名规则

- 主按钮尽量 4 到 6 个字
- 一个按钮只表达一个动作
- 同页不要出现太多语义接近的主按钮

## 8. 不建议使用的文案

以下表达不建议直接用于普通用户主界面：

- 开始执行模板
- 渲染 Prompt
- 启动图像工作流
- 选择运行模式
- 执行 Provider 路由
- 输出结构化结果
- 调整输入保真度
- 进入专家模式

原因：

- 技术味太重
- 会增加普通用户理解成本
- 破坏“轻松上手”的目标

## 9. 文档结论

普通用户版文案的核心任务，不是解释系统有多强，而是让用户感觉：

- 我知道从哪里开始
- 我知道接下来怎么做
- 我知道结果不满意时还能怎么改
- 我不用理解系统，也能顺手做出图

只要文案能做到这四点，普通用户版的可上手性就会明显提升。
