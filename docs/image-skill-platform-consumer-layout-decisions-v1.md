# 图像 Skill 平台 V1 · 普通用户版信息优先级与布局决策说明

## 文档说明

- 文档目标：说明普通用户版首页、创作页、结果页的区块排序、信息优先级、按钮主次关系和折叠策略。
- 文档定位：承接 [image-skill-platform-consumer-copy-v1.md](./image-skill-platform-consumer-copy-v1.md) 的布局决策文档。
- 当前状态：普通用户版布局原则文档。
- 更新时间：2026-05-05。

## 1. 文档目标

本文件不是重复线框稿，而是回答以下问题：

- 为什么某个区块放在前面
- 为什么某个按钮是主按钮
- 为什么某些信息默认折叠
- 为什么普通用户先看到这些，而不是那些

它的价值在于：

- 帮后续页面设计减少反复摇摆
- 帮开发理解页面结构不是随意摆放
- 帮产品在评审时有明确判断依据

## 2. 总体布局原则

普通用户版页面布局遵循以下 5 条原则：

- 先任务，后能力
- 先结果，后参数
- 先行动，后解释
- 先继续，后重来
- 先轻量，后展开

一句话：

页面应优先服务“马上开始”和“马上继续”，而不是优先服务“完整理解系统”。

## 3. 首页布局决策

## 3.1 首页第一优先级：任务入口

首页首屏必须先放任务入口，而不是：

- 大作品墙
- 参数入口
- 模板列表
- 品牌介绍长文案

原因：

- 普通用户进入站点后最先要解决的是“我从哪里开始”
- 首页的首屏如果不直接回答这个问题，用户会犹豫

因此首页第一屏顺序必须是：

1. 主问题：你现在想做什么图？
2. 任务入口
3. 最近继续创作

## 3.2 为什么“最近继续创作”要放在前半屏

原因：

- 老用户最常见行为不是重新浏览，而是继续上次任务
- 如果“最近继续创作”放太后，老用户每次都要重新找

布局原则：

- 对新用户，首屏主入口最重要
- 对老用户，最近继续创作第二重要

所以它必须高于：

- 灵感示例
- 最近结果
- 作品展示

## 3.3 为什么“热门场景”高于“灵感示例”

原因：

- 热门场景更接近任务语言
- 灵感示例更接近浏览行为
- 普通用户通常先知道任务，再被示例启发

因此首页中段顺序应是：

1. 热门场景
2. 灵感示例

而不是反过来。

## 3.4 为什么首页不先放大输入框

原因：

- 空白输入框会增加普通用户压力
- 普通用户更容易“先选任务，再补内容”

因此首页首屏应该是：

- 任务卡片 + 场景卡片

而不是：

- 一整块空白 Prompt 输入区

## 3.5 首页按钮主次关系

首页最重要的动作只有两个：

- `立即开始`
- `继续做`

其他动作如：

- `查看结果`
- `照这个做`
- `用这个风格`

都属于次级动作。

原因：

- 首页的任务是把用户推进到下一步
- 首页不是让用户停留浏览

## 4. 创作页布局决策

## 4.1 为什么创作页必须是“三栏”

建议结构：

- 左侧：输入与引导
- 中间：结果画布
- 右侧：轻量状态

原因：

- 左侧适合用户行动输入
- 中间适合承载视觉结果
- 右侧适合放不阻塞主流程的补充信息

如果只做两栏，通常会出现两个问题：

- 输入区和结果区互相抢空间
- 轻量状态和高级信息容易混入主流程

## 4.2 为什么左侧一定按“步骤顺序”组织

左侧建议顺序：

1. 你想做什么
2. 上传图片
3. 选择偏好
4. 系统补全
5. 开始生成

原因：

- 这符合普通用户的自然思路
- 先有目标，再补素材，再补方向，再开始

如果把参数和图片上传放在最上方，普通用户会先被工具感压住。

## 4.3 为什么中间结果区必须最大

原因：

- 这是一个图片产品
- 用户最在意的是结果长什么样
- 页面若过度强调左侧配置，会让人误以为这是一个专业控制台

所以视觉占比上必须保证：

- 中间结果区最大
- 左侧输入区次之
- 右侧状态区最轻

## 4.4 为什么右侧只放轻量状态，不放大量参数

原因：

- 普通用户版右侧如果堆太多内容，会造成第二个参数面板
- 右侧更适合做“你现在正在做什么”的说明

因此右侧只应放：

- 当前任务
- 当前风格
- 当前比例
- 当前状态
- 最近一次操作

而不是：

- Prompt
- Provider
- 运行模式
- 模板结构
- 参数 JSON

## 4.5 为什么“帮我补全细节”在生成前

原因：

- 这是普通用户版的核心价值点之一
- 系统能力应该在用户发起前帮他减少试错

布局上它必须放在：

- 生成按钮上方

而不能作为隐藏入口放在某个角落。

## 4.6 为什么“先试试看”是主按钮

原因：

- 对普通用户来说，主心理门槛不是“不想生成”，而是“怕做错”
- “先试试看”比“立即生成”更低压力

因此按钮层级建议是：

- 主按钮：`先试试看`
- 次按钮：`直接生成`
- 次按钮：`做 4 张方案`

## 4.7 为什么“更多设置”必须折叠

原因：

- 高级能力不能消失，但也不能主导首屏
- 普通用户只需要基本偏好即可开工

因此以下内容都应默认折叠：

- 高清输出
- 更多风格控制
- 更多生成张数控制

## 5. 结果页布局决策

## 5.1 为什么结果页先看图，再看按钮

原因：

- 用户拿到结果后的第一反应是判断“这张图对不对”
- 如果按钮先抢走注意力，会削弱结果判断

因此结果页顺序必须是：

1. 主结果大图
2. 候选图切换
3. 继续修改动作
4. 保存与导出
5. 结果说明

## 5.2 为什么“继续修改动作区”优先于“下载区”

原因：

- 这个产品的目标不是一次性下载，而是持续创作
- 普通用户大多数时候第一版不会是最终版

所以结果页应优先鼓励：

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景

而不是把“下载”做成全场最显眼按钮。

## 5.3 为什么继续修改动作只保留 6 个

原因：

- 太多动作会重新带来决策负担
- 普通用户不需要看到所有系统能力

首屏只保留 6 个，是为了覆盖 80% 高频需求：

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景
- 再生成几张
- 回到上一步重试

## 5.4 为什么“结果说明区”要放在较后位置

原因：

- 它重要，但不是第一优先级
- 用户先看图，再决定是否读解释

因此它应放在：

- 主结果区之后
- 动作区之后

它是增强理解和安全感的，不是主决策区。

## 5.5 为什么高级信息默认折叠

结果页默认不应暴露：

- Prompt
- Provider
- 版本图谱
- 参数细节

原因：

- 这些信息对普通用户来说不是继续修改的必要前提
- 会显著破坏结果页的轻体验

因此建议统一进入：

- `查看详细信息`

## 6. 普通版与进阶模式布局决策

## 6.1 为什么首页不建议直接出现进阶入口

原因：

- 首页是任务分发页
- 不是模式选择页

普通用户如果在首页就看到“高级模式 / 进阶模式”，会被提前带偏。

因此：

- 首页不强调模式切换
- 模式切换集中放在创作页和结果页

## 6.2 为什么进阶入口放右上角

原因：

- 右上角适合“非主流程但可随时进入”的动作
- 不会干扰主内容区

进阶入口不应放在：

- 首屏主按钮旁
- 任务入口卡片中
- 左侧步骤流中

## 6.3 为什么进阶模式要“展开”，而不是“跳新站”

原因：

- 用户当前上下文非常重要
- 切换模式不能意味着切换任务系统

所以更合理的是：

- 在原任务基础上展开更多能力
- 而不是跳到一套完全不同的界面重新开始

## 7. 折叠与展开策略

## 7.1 必须默认展开的内容

- 当前任务
- 核心输入
- 图片上传
- 轻量偏好
- 系统补全
- 主结果
- 继续修改动作

## 7.2 必须默认折叠的内容

- 高级参数
- Prompt 细节
- Provider 信息
- 模板内部结构
- 运行模式说明

## 7.3 可按需展开的内容

- 更多设置
- 详细信息
- 版本细节

## 8. 开发与设计上的直接含义

这份布局决策文档会直接影响后续两件事：

### 8.1 对设计

- 设计稿不能随意把重要区块换位
- 主按钮不能因为视觉美观而改变层级
- 折叠区不能被放到首屏重点位置

### 8.2 对开发

- 页面区块需要按优先级实现
- 先把主流程区块做通，再做高级折叠区
- 状态管理需要支持默认折叠与按需展开

## 9. 文档结论

普通用户版的信息优先级与布局决策，核心不是“页面怎么摆更好看”，而是：

- 用户最先需要看到什么
- 用户最先需要做什么
- 哪些内容必须先出现
- 哪些内容必须先藏起来

只要这套优先级不乱，后面的视觉设计和实际开发就更容易稳定推进。
