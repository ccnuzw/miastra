# 图像 Skill 平台 V1 · 普通用户版核心页面拆解

## 文档说明

- 文档目标：拆解普通用户版 3 张核心页面的页面目标、信息层级、核心模块与关键交互。
- 文档定位：承接 [image-skill-platform-consumer-v1.md](./image-skill-platform-consumer-v1.md) 的页面级设计文档。
- 当前状态：普通用户版页面结构文档。
- 更新时间：2026-05-05。

## 1. 页面范围

本文件只覆盖普通用户版的 3 张核心页面：

1. 首页
2. 创作页
3. 结果页

目标不是画高保真，而是先把普通用户真正会走的主路径拆清楚。

## 2. 首页

首页的任务不是展示能力，而是帮用户立刻开工。

### 2.1 页面目标

- 新用户 5 秒内知道从哪里开始
- 老用户 5 秒内回到上次任务
- 用户不需要理解“文生图 / 图生图 / 模板系统”等专业概念

### 2.2 首屏信息层级

1. 你现在想做什么图？
2. 4 到 6 个大任务入口
3. 最近继续创作
4. 热门场景

### 2.3 首屏建议模块

- 主标题：`你现在想做什么图？`
- 副标题：`一句话描述，或上传一张图，我们帮你开始`
- 大入口卡片：
  - 从文字生成图片
  - 按现有图片继续修改
  - 做商品图
  - 做海报或宣传图
  - 做头像或人像图
  - 继续上次创作

### 2.4 首页完整模块顺序

1. 主任务入口区
2. 最近继续创作区
3. 热门场景区
4. 灵感示例区
5. 极简搜索区
6. 最近结果区

### 2.5 模块拆细

#### 主任务入口区

每张卡片包含：

- 场景图标
- 简短名称
- 一句副说明
- 主按钮：`立即开始`

#### 最近继续创作区

展示 3 到 6 条最近任务。

每条卡片包含：

- 缩略图
- 任务类型
- 最近状态
- 两个按钮：`继续做`、`查看结果`

#### 热门场景区

- 商品白底图
- 商品换背景
- 活动海报
- 社媒封面
- 头像美化
- 证件照或职业照
- 去杂物
- 局部修改

#### 灵感示例区

每张图都要有动作，不只是展示。

动作建议：

- `照这个做`
- `用这个风格`
- `改成我的内容`

#### 极简搜索区

输入提示建议：

- 海报
- 白底商品图
- 换背景
- 头像

搜索结果优先返回：

- 任务入口
- 场景模板
- 最近历史

### 2.6 首页关键交互

- 点任务入口后，不先跳空白页，直接进入对应引导态创作页
- 点“继续上次创作”后，直接恢复上下文
- 点场景卡后，自动带入默认模板和示例描述
- 点示例图后，进入“仿做模式”

### 2.7 首页要隐藏的专业能力

- 不展示 Skill 分类
- 不展示模板 ID
- 不展示 Prompt
- 不展示 Provider
- 不展示模型切换
- 不展示复杂任务状态树

### 2.8 首页成功标准

- 用户首屏不迷路
- 用户第一次进入就能点一个入口开工
- 首页不是空白输入框，也不是作品墙

## 3. 创作页

创作页的任务不是让用户配参数，而是帮用户把一句需求变成第一版结果。

### 3.1 页面目标

- 让普通用户用最少输入启动任务
- 让系统主动补全缺失信息
- 让用户始终知道下一步做什么

### 3.2 页面结构

- 左侧：输入与引导
- 中间：结果画布
- 右侧：轻量状态与说明

### 3.3 左侧输入区层级

1. 任务说明
2. 图片上传
3. 偏好选择
4. 系统补全
5. 开始生成

### 3.4 左侧模块拆细

#### 任务说明

- 主输入框文案：`描述一下你想做的图片`
- 支持一句人话直接输入
- 下方给 3 条动态示例

示例：

- 做一张高级感新品海报
- 把这张图背景换成影棚
- 生成一个干净的商品白底图

#### 图片上传

根据入口动态变化：

- 从文字生成图片：参考图上传为可选
- 按现有图片继续修改：原图上传为必需

支持轻量动作：

- `上传原图`
- `再加一张参考图`
- `只改图片的一部分`

#### 偏好选择

只保留普通用户可理解的选项：

- 风格：简洁 / 高级 / 可爱 / 商业 / 写实
- 用途：商品展示 / 宣传海报 / 社媒封面 / 头像
- 比例：方图 / 竖图 / 横图

#### 系统补全

- 一个明显按钮：`帮我补全细节`
- 补全后给一句摘要：
  - `将优先保留主体，并补全背景、光线和构图`
- 如果需要追问，只问 1 到 2 个问题

#### 开始生成

主按钮：`先试试看`

次按钮：

- `直接生成`
- `做 4 张方案`

### 3.5 中间结果画布区

默认未生成前显示大占位引导。

生成后显示：

- 当前主结果大图
- 可切换其他候选结果
- 放大预览
- 生成进度

生成中状态要可视化：

- 正在理解需求
- 正在补全画面细节
- 正在生成图片

### 3.6 右侧轻量信息区

普通用户版右侧只显示：

- 当前任务：商品图 / 海报 / 改图
- 当前风格：高级感 / 简洁 / 写实
- 当前比例：方图 / 竖图 / 横图
- 当前状态：待生成 / 生成中 / 已完成
- 最近一次操作：上传了原图、选择了商品展示风格

可提供一个折叠区：

- `更多设置`

展开后才出现：

- 高清输出
- 生成张数
- 更多风格控制

### 3.7 创作页关键交互

- 用户输入一句话后，系统实时识别大致意图
- 如果识别到明确场景，自动切换到对应引导
- 用户上传图片后，文案同步变化：
  - `你想保留主体，还是主要换背景？`
- 用户不想回答时，始终有：
  - `你帮我决定`

### 3.8 创作页追问设计

追问必须短、具体、可选。

示例：

- 这张图更偏海报，还是更偏商品展示？
- 你更想保留人物，还是更想换背景？
- 要不要我帮你自动补全光线和细节？

回答方式优先用按钮，不优先用长文本。

### 3.9 创作页要隐藏的专业能力

- 不展示模板结构
- 不展示 Prompt 草稿
- 不展示运行模式
- 不展示 Provider 路由
- 不展示模型细参数
- 不展示约束字段

### 3.10 创作页成功标准

- 用户不需要研究参数也能发起一次任务
- 用户在生成前不会被复杂表单劝退
- 用户感觉系统是在帮他做图，而不是让他配工具

## 4. 结果页

结果页的任务不是结束，而是让用户顺手进入下一轮修改。

### 4.1 页面目标

- 让用户快速判断结果是否满意
- 让用户自然进入二次创作
- 让“继续改”比“重新开始”更容易

### 4.2 页面结构

1. 结果大图区
2. 候选结果切换区
3. 继续修改动作区
4. 保存与导出区
5. 结果说明区

### 4.3 结果大图区

- 当前选中结果大图居中展示
- 支持放大查看
- 支持左右切换候选图
- 如果是多张方案，切换必须顺滑直观

### 4.4 候选结果切换区

- 缩略图横排
- 当前结果高亮
- 每张结果允许：
  - 设为主结果
  - 收藏
  - 删除该候选

### 4.5 继续修改动作区

这一区是结果页核心。

建议首屏只放 6 个大动作：

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景
- 再生成几张
- 回到上一步重试

这些动作对应的是系统内部更复杂的 Skill 能力，但前台只展示普通用户看得懂的动作。

### 4.6 保存与导出区

- 下载
- 收藏
- 保存到我的作品
- 分享或复制链接

普通用户版这里不优先展示“导出 Prompt”。

### 4.7 结果说明区

给普通用户一段简单解释，而不是技术详情。

示例：

- 当前是“高级感商品海报”风格
- 已优先保留商品主体和整体构图
- 如果不满意，可以继续换背景或调整风格

如果系统做了重要保护，也应明确告诉用户：

- 已尽量保留人物特征
- 已按原图比例输出
- 已优先保留标签和材质

### 4.8 结果页关键交互

- 点击“更像这一张”后，不跳走，直接基于当前结果生成下一轮
- 点击“只改这一部分”后，进入局部编辑态，而不是全新页面重开
- 点击“保留主体换背景”后，自动带入当前结果作为基础图
- 点击“回到上一步重试”时，恢复上一轮输入，不让用户重填

### 4.9 结果页隐藏的专业能力

- 默认不展示完整 Prompt
- 默认不展示任务原始 JSON
- 默认不展示 Provider / model
- 默认不展示版本链图谱
- 默认不展示高级生成参数

这些内容可进入：

- `查看详细信息`

### 4.10 结果页成功标准

- 用户拿到图后，不会不知道下一步怎么改
- 用户更愿意点“继续改”，而不是重新开始
- 结果页像一个创作节点，而不是下载终点

## 5. 三页之间的关系

普通用户版三页必须形成一个非常顺的闭环：

- 首页负责“开始”
- 创作页负责“生成第一版”
- 结果页负责“继续变好”

这三页共同组成的体验应是：

`我有想法 -> 我试一版 -> 我继续改到满意`

## 6. 文档结论

普通用户版的 3 张核心页面不应按“能力清单”组织，而应按“任务推进”组织。

也就是：

- 首页不是作品陈列
- 创作页不是参数后台
- 结果页不是下载终点

它们共同服务的是普通用户最自然的一条路径：

先开始，再试一版，再顺手改到满意。
