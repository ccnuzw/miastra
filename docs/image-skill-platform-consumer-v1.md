# 图像 Skill 平台 V1 · 普通用户版

## 文档说明

- 文档目标：定义面向普通用户的前台界面、任务入口和创作流程。
- 文档定位：承接 [image-skill-platform-v1-prd.md](./image-skill-platform-v1-prd.md) 的用户侧产品设计文档。
- 当前状态：普通用户版产品规划主文档。
- 更新时间：2026-05-05。

## 1. 核心定位

普通用户版不是“专业创作工具的简化壳”，而是一套重新按普通用户认知设计的做图产品层。

一句话定位：

一个会帮用户把想法变成图片的智能做图助手。

产品目标：

- 看得懂
- 开始快
- 少输入
- 少决策
- 出结果后容易继续改

## 2. 核心原则

普通用户版必须遵循以下原则：

- 用户只需要知道“我要做什么图”，不需要知道系统内部如何工作。
- 底层可以很专业，前台语言必须生活化。
- 前台按任务设计，后台按 Skill 设计。
- 复杂能力可以保留，但默认必须隐藏。
- 用户体验必须任务导向，而不是参数导向。

一句话总结：

专业能力内置，用户体验去专业化。

## 3. 普通用户版不直接暴露的概念

以下概念可以存在于系统内部或进阶模式中，但不应成为普通用户主界面语言：

- Skill
- Prompt Draft
- Rendered Prompt
- Provider
- Runtime Mode
- Constraints
- Template DSL
- Input Fidelity
- Output Compression

普通用户应看到的，是更生活化、更动作化的表达：

- 做什么图
- 参考哪张图
- 想改哪里
- 想要什么感觉
- 帮我补全
- 先试试看
- 不满意再改

## 4. 普通用户总流程

普通用户版建议只保留 4 步主流程：

1. 选任务
2. 说需求
3. 看结果
4. 继续改

展开后是：

1. 用户进入首页，选择“我要做什么”。
2. 系统引导用户输入一句话或上传一张图。
3. 系统自动补全细节并生成结果。
4. 用户对结果做二次操作，而不是回到起点重来。

## 5. 首页结构

首页只解决两个问题：

- 新用户怎么快速开始
- 老用户怎么快速继续

### 5.1 顶部主入口

首页最上方先问一个问题：

`你现在想做什么图？`

下面给 4 到 6 个大入口卡片：

- 从文字生成图片
- 按现有图片继续修改
- 做商品图
- 做海报或宣传图
- 做头像或人像图
- 继续上次创作

要求：

- 主入口使用任务语言，而不是能力语言。
- 不直接把“文生图”“图生图”“局部重绘”作为首屏主按钮。

### 5.2 热门场景区

普通用户常常不知道如何描述，因此首页必须提供场景入口。

建议首批场景：

- 商品白底图
- 商品换背景
- 活动海报
- 社媒封面
- 头像美化
- 证件照或职业照
- 去杂物或去文字
- 局部修改

### 5.3 最近继续创作

这块应放在首页前半屏，优先级高于灵感墙。

展示内容：

- 最近做过的图
- 正在生成中的任务
- 上次未完成的编辑
- 最近收藏的结果

每条卡片只保留两个主要动作：

- 继续做
- 查看结果

### 5.4 灵感示例区

普通用户高度依赖示例。

每张示例卡片都应提供直接动作：

- 照这个做一张
- 用这个风格
- 改成我的内容

### 5.5 极简搜索区

当用户目的明确时，可以直接输入：

- 海报
- 白底商品图
- 换背景
- 头像

搜索结果优先返回：

- 任务入口
- 场景模板
- 最近相关历史

## 6. 创作页结构

创作页不应是参数面板，而应是引导式做图页。

设计目标：

- 让用户尽量少想
- 让系统尽量多补
- 让复杂能力隐形运行

建议结构：

- 左侧：输入与引导
- 中间：结果区
- 右侧：轻量信息区

### 6.1 左侧输入区

普通用户版左侧输入应按如下顺序组织：

#### 第一步：你想做什么

- 一句话输入框
- 支持直接说人话

示例：

- 做一张新品海报
- 把这张图背景换成影棚
- 生成一个干净的商品白底图

#### 第二步：上传图片

如果当前任务需要图片：

- 上传原图
- 上传参考图
- 可选“只改这张图的一部分”

#### 第三步：选择偏好

只保留普通用户可以理解的选项：

- 风格：简洁 / 高级 / 可爱 / 商业 / 写实
- 用途：商品展示 / 宣传海报 / 社媒封面 / 头像
- 画面比例：方图 / 竖图 / 横图

#### 第四步：系统补全

提供一个明确动作：

`帮我补全细节`

前台解释应简单直接，例如：

- 系统将优先保留主体，并补全背景、光线和构图细节

#### 第五步：开始生成

主按钮文案建议：

- 立即生成
- 先试试看
- 做 4 张方案

其中“先试试看”更适合普通用户首次尝试。

### 6.2 中间结果区

中间结果区必须是整个页面的视觉中心。

普通用户最关心的是：

- 图出来了吗
- 好不好看
- 下一步能怎么改

结果区建议支持：

- 单张大图预览
- 多结果横向切换
- 点击放大
- 生成中状态可视化

### 6.3 右侧轻量信息区

普通用户版右侧不要展示复杂参数墙。

建议只展示：

- 当前任务类型
- 当前风格倾向
- 当前尺寸
- 当前状态
- 最近一次操作记录

## 7. 动态追问设计

系统可以很智能，但追问必须像“帮忙补全”，不能像“填写问卷”。

### 7.1 追问原则

- 每次最多问 1 到 2 个问题
- 问题必须短
- 问题必须具体
- 问题必须容易回答
- 尽量提供按钮选项，不强迫长文本输入

### 7.2 示例

不推荐：

- 你希望整体视觉风格、构图逻辑和氛围如何？

推荐：

- 这张图更偏海报，还是更偏商品展示？
- 你更想保留人物，还是更想换背景？
- 要不要我帮你自动补全光线和细节？

### 7.3 回答方式

普通用户版建议支持两种回答方式：

- 一键选项
- 简短补充输入

并始终提供一个明确入口：

`你帮我决定`

## 8. 结果页设计

结果页不应只是“看图 + 下载”，而应该是“二次创作的开始页”。

### 8.1 最重要的 6 个动作

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景
- 再生成几张
- 回到上一步重试

这些动作对普通用户可理解，对系统内部则映射到更复杂的 Skill 能力。

### 8.2 结果页结构建议

- 上方：大图预览 + 多结果切换
- 下方第一排：继续修改动作
- 下方第二排：下载、收藏、保存
- 侧边或底部：这张图的简单说明

说明示例：

- 当前是“高级感商品海报”风格
- 已优先保留商品主体与构图
- 如果不满意，可以继续换背景或调整风格

## 9. 历史页设计

历史页对普通用户的价值是“找回上次做过的图”，不是“查看复杂参数”。

### 9.1 重点功能

- 最近做过
- 最近下载过
- 最近收藏过
- 最近没做完的
- 同类任务聚合

### 9.2 每张卡片建议动作

- 继续做
- 查看结果
- 再做一版

要求：

- 默认不展示复杂版本树
- 版本关系可放到详情页中

## 10. 普通用户版与专业版切换

建议采用双层产品模式，而不是在同一页面堆很多参数。

### 10.1 默认模式：普通用户版

特点：

- 用任务语言
- 少参数
- 多引导
- 多示例
- 多一键操作
- 系统自动补全

### 10.2 进阶模式：专业创作版

特点：

- 可看模板
- 可改 Prompt
- 可调高级参数
- 可看执行模式
- 可切 Provider
- 可导出 Prompt
- 可看完整版本信息

### 10.3 切换策略

- 默认所有用户进入普通模式
- 在工作台右上角提供“进入进阶模式”
- 不建议首次进入时让用户主动选择模式
- 普通模式和专业模式共享同一套任务与历史资产

## 11. 前台文案原则

### 11.1 文案原则

- 用动作词，不用技术词
- 用“你想做什么”，不用“请选择能力”
- 用“系统帮你补全”，不用“请配置参数”
- 用“继续改”，不用“再次调用”

### 11.2 推荐表达

- `从文字生成图片` 优于 `文生图`
- `按这张图继续修改` 优于 `图生图`
- `只改这一部分` 优于 `局部重绘`
- `换成新背景` 优于 `背景替换工作流`
- `帮我优化描述` 优于 `Prompt Rewrite`

专业概念可以存在，但不应成为普通用户主界面的主语言。

## 12. 普通用户版一期重点

建议优先落地以下 6 件事：

- 首页改成任务入口页，不再是能力堆叠页
- 创作页改成引导式流程，不再强调复杂参数
- 系统内置模板与追问，但前台不暴露专业术语
- 结果页强化“继续改”的动作体系
- 历史页强化“找回与继续”，弱化专业信息展示
- 预留“进入进阶模式”的入口，但不默认展开

## 13. 文档结论

普通用户版真正的关键，不是减少能力，而是重新包装能力。

也就是：

- 底层仍然是图像 Skill 平台
- 前台表现成智能做图助手

这才是一款面向普通用户的线上产品应该有的形态。
