# 图像 Skill 平台 V1 · 普通用户版线框级信息结构

## 文档说明

- 文档目标：把普通用户版 3 张核心页面继续拆到线框级信息结构，作为后续原型与界面设计依据。
- 文档定位：承接 [image-skill-platform-consumer-pages-v1.md](./image-skill-platform-consumer-pages-v1.md) 的页面骨架文档。
- 当前状态：普通用户版线框级结构文档。
- 更新时间：2026-05-05。

## 1. 文档范围

本文件覆盖以下 3 张页面：

1. 首页
2. 创作页
3. 结果页

这里的“线框级”含义是：

- 明确页面区域划分
- 明确信息优先级
- 明确主按钮和次按钮
- 明确每个区块的交互入口
- 不进入视觉风格、配色和高保真组件细节

## 2. 首页线框结构

### 2.1 首页目标

- 让新用户快速开工
- 让老用户快速回到上次任务
- 首屏不暴露专业术语

### 2.2 首页整体结构

从上到下建议分为 6 个区块：

1. 顶部导航区
2. 首屏主任务区
3. 最近继续创作区
4. 热门场景区
5. 灵感示例区
6. 搜索与最近结果区

### 2.3 首页线框草图说明

```text
+----------------------------------------------------------------------------------+
| 顶部导航：Logo | 我的作品 | 历史记录 | 收藏 | 账号                               |
+----------------------------------------------------------------------------------+
| 首屏主任务区                                                                    |
| 标题：你现在想做什么图？                                                         |
| 副标题：一句话描述，或上传一张图，我们帮你开始                                   |
| [从文字生成图片] [按现有图片继续修改] [做商品图] [做海报/宣传图] [做头像/人像图] |
| [继续上次创作]                                                                   |
+----------------------------------------------------------------------------------+
| 最近继续创作                                                                     |
| [任务卡 1] [任务卡 2] [任务卡 3]                                                 |
+----------------------------------------------------------------------------------+
| 热门场景                                                                         |
| [商品白底图] [商品换背景] [活动海报] [社媒封面] [头像美化] [去杂物] [局部修改]   |
+----------------------------------------------------------------------------------+
| 灵感示例                                                                         |
| [示例卡 1] [示例卡 2] [示例卡 3] [示例卡 4]                                      |
+----------------------------------------------------------------------------------+
| 搜索：我想做_____                                                                |
| 最近结果： [结果卡 1] [结果卡 2] [结果卡 3]                                      |
+----------------------------------------------------------------------------------+
```

### 2.4 顶部导航区

建议元素：

- Logo
- 我的作品
- 历史记录
- 收藏
- 账号入口

要求：

- 顶部不要放“模型”“高级设置”“Prompt”等专业入口
- 进阶入口不应在首页主导航显著暴露

### 2.5 首屏主任务区

这是首页最重要的区块。

建议内容：

- 主标题：`你现在想做什么图？`
- 副标题：`一句话描述，或上传一张图，我们帮你开始`
- 6 张任务入口卡片

每张卡片包含：

- 简短标题
- 一句用途说明
- 主按钮：`立即开始`

建议卡片标题：

- 从文字生成图片
- 按现有图片继续修改
- 做商品图
- 做海报或宣传图
- 做头像或人像图
- 继续上次创作

### 2.6 最近继续创作区

建议信息结构：

- 区块标题：`继续上次做到一半的内容`
- 横向卡片列表

每张卡片包含：

- 缩略图
- 任务名
- 当前状态
- 最近更新时间
- 主按钮：`继续做`
- 次按钮：`查看结果`

### 2.7 热门场景区

建议信息结构：

- 区块标题：`常用场景`
- 2 行卡片网格

每张场景卡片只表达一个动作：

- 商品白底图
- 商品换背景
- 活动海报
- 社媒封面
- 头像美化
- 证件照或职业照
- 去杂物或去文字
- 局部修改

卡片点击后直接进入带默认引导的创作页。

### 2.8 灵感示例区

建议信息结构：

- 区块标题：`看看别人怎么做`
- 作品卡片流

每张卡片包含：

- 示例图
- 一句场景说明
- 主按钮：`照这个做`
- 次按钮：`用这个风格`

### 2.9 搜索与最近结果区

建议信息结构：

- 搜索输入框：`我想做……`
- 热门搜索建议
- 最近结果缩略图区

最近结果卡片包含：

- 缩略图
- 场景标签
- 主按钮：`继续做`

### 2.10 首页主按钮优先级

最高优先级按钮：

- `立即开始`
- `继续做`

次优先级按钮：

- `查看结果`
- `照这个做`

首页上不应出现过多平级按钮。

## 3. 创作页线框结构

### 3.1 创作页目标

- 让用户快速提交一次创作任务
- 让复杂能力隐藏在引导之下
- 让用户始终知道下一步怎么做

### 3.2 创作页整体结构

建议采用三栏布局：

1. 左侧输入与引导区
2. 中间主画布与结果区
3. 右侧轻量状态区

### 3.3 创作页线框草图说明

```text
+------------------------------------------------------------------------------------------------------+
| 顶部：返回 | 当前任务标题 | 保存草稿 | 进入进阶模式                                                   |
+------------------------------------------------------------------------------------------------------+
| 左侧输入区                | 中间结果区                                   | 右侧状态区             |
|--------------------------|----------------------------------------------|------------------------|
| 1. 你想做什么            | 结果预览区 / 占位引导                        | 当前任务               |
| 输入框                    | [未生成时显示引导]                           | 当前风格               |
| 示例提示                  | [生成后显示主结果大图]                       | 当前比例               |
|                          |                                              | 当前状态               |
| 2. 上传图片              | 候选结果条                                   | 最近一次操作           |
| [上传原图]               | [图1] [图2] [图3]                            |                        |
| [再加一张参考图]         |                                              | 更多设置（折叠）       |
| [只改这一部分]           |                                              |                        |
|                          |                                              |                        |
| 3. 选择偏好              | 生成中状态                                   |                        |
| 风格 / 用途 / 比例        | 正在理解需求 / 补全细节 / 生成图片            |                        |
|                          |                                              |                        |
| 4. 系统补全              |                                              |                        |
| [帮我补全细节]           |                                              |                        |
| 简短摘要                  |                                              |                        |
|                          |                                              |                        |
| 5. 开始生成              |                                              |                        |
| [先试试看] [直接生成]    |                                              |                        |
| [做4张方案]              |                                              |                        |
+------------------------------------------------------------------------------------------------------+
```

### 3.4 顶部条

建议包含：

- 返回
- 当前任务标题
- 保存草稿
- 进入进阶模式

要求：

- “进入进阶模式”存在，但不抢主流程视觉焦点
- 顶部不要出现模型切换或 Provider 暴露项

### 3.5 左侧输入与引导区

分为 5 个步骤块：

1. 你想做什么
2. 上传图片
3. 选择偏好
4. 系统补全
5. 开始生成

#### 3.5.1 你想做什么

元素：

- 大输入框
- 3 条动态示例
- 可选语音或快捷填充入口

主按钮：

- 无单独提交按钮，继续往下即可

#### 3.5.2 上传图片

元素：

- 上传原图按钮
- 增加参考图按钮
- 只改这一部分入口

规则：

- 根据任务入口动态调整必填和可选状态

#### 3.5.3 选择偏好

元素：

- 风格单选组
- 用途单选组
- 比例单选组

规则：

- 每组尽量控制在 3 到 5 项内
- 不展开技术参数

#### 3.5.4 系统补全

元素：

- 按钮：`帮我补全细节`
- 摘要说明区
- 如有追问，插入 1 到 2 条问题卡片

追问卡片建议结构：

- 问题标题
- 2 到 4 个选项按钮
- 次按钮：`你帮我决定`

#### 3.5.5 开始生成

主按钮：

- `先试试看`

次按钮：

- `直接生成`
- `做 4 张方案`

设计原则：

- 普通用户默认鼓励“先试试看”

### 3.6 中间结果区

未生成前：

- 显示大尺寸占位区
- 显示一句引导：
  - `填写左侧内容后，这里会出现结果`

生成后：

- 主结果大图
- 候选结果条
- 点击放大
- 切换结果

生成中：

- 不只显示 loading
- 分阶段显示状态：
  - 正在理解需求
  - 正在补全细节
  - 正在生成图片

### 3.7 右侧轻量状态区

建议内容：

- 当前任务
- 当前风格
- 当前比例
- 当前状态
- 最近一次操作

折叠区：

- `更多设置`

展开后才显示：

- 高清输出
- 生成张数
- 更多风格控制

### 3.8 创作页主按钮优先级

最高优先级：

- `先试试看`

次优先级：

- `帮我补全细节`
- `直接生成`

辅助优先级：

- `做 4 张方案`
- `你帮我决定`

## 4. 结果页线框结构

### 4.1 结果页目标

- 让用户快速判断结果是否满意
- 让用户自然进入二次创作
- 让结果页成为继续创作的节点，而不是结束页

### 4.2 结果页整体结构

建议分为 5 个区块：

1. 主结果展示区
2. 候选结果切换区
3. 继续修改动作区
4. 保存与导出区
5. 结果说明区

### 4.3 结果页线框草图说明

```text
+--------------------------------------------------------------------------------------------------+
| 顶部：返回创作页 | 当前结果标题 | 保存到作品 | 收藏                                               |
+--------------------------------------------------------------------------------------------------+
| 主结果展示区                                                                                    |
| [当前大图预览]                                                                                  |
+--------------------------------------------------------------------------------------------------+
| 候选结果切换区                                                                                  |
| [缩略图1] [缩略图2] [缩略图3] [缩略图4]                                                         |
+--------------------------------------------------------------------------------------------------+
| 继续修改动作区                                                                                  |
| [更像这一张] [换个风格] [只改这一部分] [保留主体换背景] [再生成几张] [回到上一步重试]          |
+--------------------------------------------------------------------------------------------------+
| 保存与导出区                                                                                    |
| [下载] [收藏] [保存到我的作品] [分享]                                                           |
+--------------------------------------------------------------------------------------------------+
| 结果说明区                                                                                      |
| 当前风格说明                                                                                    |
| 系统保护说明                                                                                    |
| 下一步建议                                                                                      |
+--------------------------------------------------------------------------------------------------+
```

### 4.4 顶部条

建议内容：

- 返回创作页
- 当前结果标题
- 保存到作品
- 收藏

不建议普通用户首屏就看到：

- Prompt
- Provider
- 模型信息

### 4.5 主结果展示区

要求：

- 当前选中结果大图居中展示
- 支持点击放大
- 支持左右切换
- 视觉上必须让用户先看图，再决定动作

### 4.6 候选结果切换区

每张缩略图建议支持：

- 点击切换为主结果
- 收藏该结果
- 删除该候选

当前选中的结果应明显高亮。

### 4.7 继续修改动作区

这是结果页最重要的区域。

建议首屏只放 6 个大动作：

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景
- 再生成几张
- 回到上一步重试

设计原则：

- 每个动作都必须是普通用户能一眼看懂的表达
- 不出现技术性按钮命名

### 4.8 保存与导出区

建议按钮：

- 下载
- 收藏
- 保存到我的作品
- 分享

普通用户版这里不优先展示“导出 Prompt”或“复制参数”。

### 4.9 结果说明区

建议内容：

- 当前风格说明
- 系统保护说明
- 下一步建议

示例：

- 当前是“高级感商品海报”风格
- 已优先保留商品主体和构图
- 如果不满意，可以继续换背景或调整风格

### 4.10 结果页关键交互

- 点击“更像这一张”后，直接基于当前结果生成下一轮
- 点击“只改这一部分”后，进入局部编辑态
- 点击“保留主体换背景”后，自动带入当前结果为基础图
- 点击“回到上一步重试”时，恢复上一轮输入

### 4.11 结果页主按钮优先级

最高优先级：

- `更像这一张`
- `只改这一部分`

次优先级：

- `换个风格`
- `保留主体换背景`
- `再生成几张`

辅助优先级：

- `下载`
- `收藏`
- `保存到我的作品`

## 5. 三页之间的状态流

普通用户版 3 张页面的状态流应如下：

### 5.1 首页 -> 创作页

- 用户从任务入口进入
- 系统自动带入任务语境
- 若有模板或场景预设，在创作页静默填充

### 5.2 创作页 -> 结果页

- 用户发起生成后，先留在创作页看到状态
- 当第一版结果可用后，进入结果页或切换为结果态

### 5.3 结果页 -> 创作页

- 用户点击继续修改动作
- 系统保留当前结果作为上下文
- 回到带状态的创作页，而不是清空重开

### 5.4 结果页 -> 首页

- 用户保存结果后可回到首页
- 首页“最近继续创作”和“最近结果”应立即反映刚才的任务

## 6. 文档结论

线框级信息结构的核心目标，是让普通用户版 3 张页面形成一条非常顺的主路径：

`我想做什么 -> 我怎么开始 -> 我怎么继续改`

只要这条路径成立，后面再进入原型和视觉设计时，专业能力就有机会被很好地隐藏在系统内部。
