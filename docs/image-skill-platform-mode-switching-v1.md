# 图像 Skill 平台 V1 · 普通用户版与专业版切换策略

## 文档说明

- 文档目标：定义普通用户版与专业版的切换方式、能力边界、信息暴露层级与状态共享规则。
- 文档定位：承接 [image-skill-platform-consumer-wireframes-v1.md](./image-skill-platform-consumer-wireframes-v1.md) 的模式边界文档。
- 当前状态：双模式产品策略文档。
- 更新时间：2026-05-05。

## 1. 文档结论先行

本产品应采用“双层产品”而不是“单页堆叠高级参数”的策略。

推荐模式：

- 默认模式：普通用户版
- 进阶模式：专业创作版

核心原则：

- 普通用户优先开工，不优先理解系统
- 专业用户优先控制力，不优先极简引导
- 两种模式共享同一套任务、历史、资产和结果

一句话总结：

前台体验双层，底层能力单层。

## 2. 为什么必须双模式

如果只做普通用户版，问题是：

- 高级用户会觉得能力不够透明
- 无法满足 Prompt、参数、Provider、模板层面的控制诉求
- 无法支持更深的复用与调试

如果只做专业版，问题是：

- 普通用户会被专业术语和复杂参数吓退
- 首次成功率会下降
- 用户不知道从哪里开始，也不知道结果不满意时怎么继续改

因此最佳解法不是二选一，而是：

- 用普通用户版承接大部分人
- 用专业版承接高控制需求

## 3. 双模式的总体策略

### 3.1 默认进入策略

- 所有新用户默认进入普通用户版
- 所有未显式切换过的老用户默认继续停留在上次模式
- 不建议首次进入时让用户先选模式

### 3.2 切换入口策略

普通用户版中提供一个低干扰入口：

- `进入进阶模式`

专业版中提供一个回退入口：

- `返回简洁模式`

要求：

- 切换入口存在，但不抢普通用户主流程焦点
- 不用“专家模式”“开发者模式”这类压迫感强的命名

### 3.3 模式本质

普通用户版是“任务导向视图”。

专业版是“控制导向视图”。

两者不是两套不同产品，而是同一任务系统的两种表现层。

## 4. 两种模式的角色定义

### 4.1 普通用户版

普通用户版解决的问题：

- 我想做什么图
- 我应该怎么开始
- 我不懂参数怎么办
- 我不满意时下一步怎么改

普通用户版优先级：

- 任务理解
- 场景入口
- 系统补全
- 一键继续修改
- 结果解释

### 4.2 专业版

专业版解决的问题：

- 我想更精确控制结果
- 我想查看和修改 Prompt
- 我想管理模板和参数
- 我想指定 Provider 或模型
- 我想复用历史资产和版本链

专业版优先级：

- 高级参数
- 模板与变体
- Prompt 透明度
- Provider 控制
- 资产复用

## 5. 信息暴露边界

这是双模式策略最关键的部分。

### 5.1 普通用户版默认不暴露

- Skill
- Template DSL
- Prompt Draft
- Rendered Prompt
- Runtime Mode
- Provider
- Model 细分选择
- Input Fidelity
- Output Compression
- 复杂约束字段
- 版本图谱结构

### 5.2 普通用户版允许轻量出现

- 当前风格
- 当前用途
- 当前比例
- 当前状态
- 系统已帮你补全
- 已优先保留主体 / 背景 / 标签等解释性说明

### 5.3 专业版允许完整暴露

- 模板名称与来源
- Prompt 内容
- 结构化参数
- 更多高级设置
- Provider 选择
- 运行模式信息
- 历史参数复制
- 版本关系细节

## 6. 能力边界划分

### 6.1 普通用户版能力清单

普通用户版应保留：

- 从文字生成图片
- 按现有图片继续修改
- 上传原图 / 参考图
- 风格、用途、比例等轻量偏好
- 一键补全细节
- 生成 1 张或多张方案
- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景
- 再生成几张
- 回到上一步重试
- 历史找回与继续创作

### 6.2 专业版新增能力

专业版在普通版基础上新增：

- 查看模板
- 查看 Prompt
- 编辑 Prompt
- 调整高级参数
- 指定 Provider
- 切换模型
- 导出 Prompt
- 复制任务参数
- 查看完整版本链
- 更精细的生成控制

### 6.3 不建议普通用户版直接继承的能力

- 复杂模板结构编辑
- 明确的 Prompt 文本调试
- 多层参数面板
- Provider 多路切换
- 过细的模型与质量控制

这些能力一旦默认暴露，会显著破坏普通用户体验。

## 7. 页面级切换策略

### 7.1 首页

首页默认始终使用普通用户版结构。

理由：

- 首页的目标是分发任务
- 首页不适合作为高级能力面板

专业用户即使存在，也应从普通首页进入，然后在创作页切换到进阶模式。

### 7.2 创作页

创作页是最核心的切换页面。

建议：

- 普通用户版默认展示引导式做图页
- 右上角提供 `进入进阶模式`
- 切换后保留当前任务上下文，不刷新任务，不丢输入

### 7.3 结果页

结果页也需要支持双模式。

普通用户版结果页：

- 以“继续改”为主

专业版结果页：

- 增加 Prompt、模板、参数、Provider 和版本细节查看能力

### 7.4 历史页

普通用户版历史页：

- 重点是找回和继续

专业版历史页：

- 重点是筛选、复用和查看任务细节

## 8. 切换时的状态要求

模式切换不能破坏用户当前上下文。

### 8.1 切换时必须保留

- 当前任务输入
- 已上传图片
- 当前候选结果
- 已完成追问
- 当前风格和用途选择
- 当前生成历史

### 8.2 切换时可以新增显示

- Prompt 面板
- 模板与参数面板
- Provider 面板
- 版本链细节

### 8.3 切换时绝不能发生

- 清空当前任务
- 丢失当前上传素材
- 让用户重新填写步骤
- 重新发起生成

## 9. 升级到专业版的触发时机

普通用户不应该一上来就被引导进专业版。  
更合理的是在特定节点“需要时再升级”。

### 9.1 建议触发点

- 用户多次重试后仍不满意
- 用户明确想修改更细节的结果
- 用户想看系统到底是怎么生成的
- 用户想复用这套配置
- 用户已经多次使用并表现出较高熟练度

### 9.2 建议触发文案

不要用：

- 切换到专家模式
- 打开高级控制台

建议用：

- 需要更多控制？进入进阶模式
- 想更细地调整这张图？打开更多设置
- 想查看完整设置和生成细节？进入进阶模式

## 10. 从专业版回到普通版

专业版用户也可能只想快速继续做图，因此应支持回退。

### 10.1 回退要求

- 不清空数据
- 不清空结果
- 只收起复杂能力
- 保留当前任务状态

### 10.2 回退后的表现

- 高级参数面板折叠
- Prompt 面板隐藏
- 页面回到普通用户能理解的动作层

## 11. 权限与产品运营视角

双模式不一定意味着权限隔离，但可以意味着产品策略分层。

### 11.1 默认建议

- 所有用户都可看到普通版
- 所有用户都可进入专业版

### 11.2 后续可选策略

- 将某些高级能力放入专业版权益
- 将某些高级导出能力作为付费能力
- 将 BYOK、模板复用、批量控制放入更高权益层

但 V1 不建议把“是否能进入专业版”作为强权限门槛。

## 12. 实现层建议

双模式在实现上不应拆成两套完全独立页面。

建议做法：

- 共享任务状态
- 共享资产与历史
- 共享结果与版本
- 在同一页面框架下按模式控制区块显隐
- 只有必要时局部切换布局密度与信息层级

也就是说：

- 数据层一套
- 任务层一套
- 能力层一套
- 表现层两套

## 13. 产品风险

### 13.1 风险一

普通版和专业版做成两套割裂流程。

结果：

- 用户切换成本高
- 状态同步复杂
- 开发维护成本高

### 13.2 风险二

普通版为了“简单”而过度阉割。

结果：

- 一旦用户不满意结果，就不知道怎么继续

### 13.3 风险三

专业版入口过于显眼。

结果：

- 普通用户被提前暴露给复杂能力

### 13.4 风险四

专业版只是普通版加更多字段，没有真正的控制力提升。

结果：

- 高级用户仍然不满意

## 14. V1 建议落地结论

V1 建议按以下策略执行：

- 首页固定使用普通用户版
- 创作页和结果页支持“进入进阶模式”
- 模式切换不丢上下文
- 普通版主打任务语言和动作语言
- 专业版补充模板、Prompt、Provider 和高级参数能力

## 15. 文档结论

双模式产品的关键不是做两套产品，而是做一套底层能力、两层前台表达。

即：

- 普通用户版负责降低门槛
- 专业版负责提升控制力

只要这条边界清楚，后续页面、交互和实现就不会互相打架。
