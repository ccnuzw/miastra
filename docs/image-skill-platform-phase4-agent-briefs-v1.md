# 图像 Skill 平台 V1 · 第四阶段多 Agent 任务说明书

## 文档说明

- 文档目标：把第四阶段 backlog 继续拆成可直接分发给多会话 / 多 Agent 的任务说明书。
- 文档定位：承接 [image-skill-platform-phase4-backlog-v1.md](./image-skill-platform-phase4-backlog-v1.md) 的执行分发文档。
- 当前状态：第四阶段多 Agent 执行说明文档。
- 更新时间：2026-05-05。

## 1. 使用方式

这份文档继续沿用前三阶段的分发方式，但目标已经切换到“结构统一和机制扩展”。

推荐使用方式：

1. 继续以当前总分支 `feature/image-skill-platform-v1` 作为集成分支
2. 每个 Agent 从总分支切自己的第四阶段子分支
3. 每个 Agent 严格按自己的负责范围改动
4. 每个 Agent 完成后先自检，再合回总分支
5. 总分支统一做第四阶段集成验证

## 2. 第四阶段分工原则

- 先压统一 schema，再做机制扩展
- 先增强共享对象，再扩展更多场景
- 按文件边界拆，但要明确共享场景语义边界
- 继续避免多个 Agent 同时深改同一个高冲突文件
- 第四阶段要特别避免“再造一层新抽象，但没有真正落到主流程”

## 3. Agent 总览

| Agent | 任务名称 | 核心目标 | 优先级 | 是否阻塞别人 |
|---|---|---|---|---|
| Agent A4 | 统一场景 Schema 深化 | 让模板、追问、执行、版本共享更统一的对象和字段命名 | P0 | 部分 |
| Agent B4 | 轻量追问配置化 | 让追问场景、步骤和默认值更容易扩展 | P0 | 部分 |
| Agent C4 | 版本链生产流增强 | 让多轮继续、重试、分叉具备更稳定的工作流心智 | P0 | 部分 |
| Agent D4 | 专业版高频复用增强 | 让专业版更明确服务结构复用、重跑和派生 | P1 | 否 |
| Agent E4 | 模板向 Skill 入口演进 | 让模板从结构入口进一步走向带执行意图的 Skill 入口 | P1 | 部分 |

## 4. Agent A4：统一场景 Schema 深化

### 4.1 任务目标

让模板结构、追问结果、执行参数、版本上下文共享更稳定的场景对象和字段命名，减少多层适配与语义分裂。

### 4.2 负责范围

- `src/features/prompt-templates/*`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`
- `src/features/generation/*`
- `src/features/works/workReplay.ts`

### 4.3 允许目标

- 定义统一场景对象与字段命名约束
- 对齐模板 schema、追问快照、执行参数、版本上下文
- 清理重复字段和映射断点
- 为后续持久化扩展预留稳定接口

### 4.4 不要做的事

- 不要顺手做更多页面改版
- 不要引入新的大而全中台抽象
- 不要回滚第三阶段已有结构字段
- 不要做后端 schema 重构

### 4.5 交付标准

- 模板、追问、执行、版本开始共享更稳定的场景字段
- 显著减少命名不一致和多层适配

### 4.6 自测清单

- 检查模板、追问、结果动作、回流链中的核心字段命名
- 验证相同场景在多个入口下语义一致
- 确认不引入新的命名分裂

### 4.7 建议子分支名

- `feature/phase4-scene-schema`

## 5. Agent B4：轻量追问配置化

### 5.1 任务目标

让当前追问从少量手工场景能力升级成更容易扩展的配置机制。

### 5.2 负责范围

- `src/features/studio-home/*`
- `src/features/studio-consumer/*`
- `src/features/prompt-templates/*`
- `src/pages/app/StudioPage.tsx`

### 5.3 允许目标

- 抽离追问配置结构
- 让模板字段驱动追问步骤和默认值
- 扩展到更多高频场景，但控制范围
- 强化追问结果到 Prompt 和结果动作的映射闭环

### 5.4 不要做的事

- 不要做复杂自由对话追问
- 不要重构专业版控制区
- 不要大范围重写模板 schema
- 不要回退成硬编码散点追问

### 5.5 交付标准

- 追问定义更清晰可扩展
- 新场景接入追问的成本明显下降
- 追问结果对工作台和结果动作的影响更稳定

### 5.6 自测清单

- 进入多个高频场景触发追问
- 检查追问配置是否可复用
- 检查追问结果到输入区、生成态、结果动作的映射是否稳定

### 5.7 建议子分支名

- `feature/phase4-guided-flow-config`

## 6. Agent C4：版本链生产流增强

### 6.1 任务目标

让版本链更稳定支持多轮继续、重试、分叉和派生，而不是只靠轻量动作语义。

### 6.2 负责范围

- `src/features/works/workReplay.ts`
- `src/pages/app/WorksPage.tsx`
- `src/pages/app/TasksPage.tsx`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`

### 6.3 允许目标

- 增强父节点、祖先节点和当前节点摘要
- 强化多轮回流与派生的来源说明
- 强化版本链与模板、追问、参数之间的对应关系
- 优化普通版和专业版中的多轮版本消费体验

### 6.4 不要做的事

- 不要做复杂版本图谱
- 不要引入项目级版本树系统
- 不要重构模板结构层
- 不要开始做完整 diff 面板

### 6.5 交付标准

- 多轮版本链更容易理解
- 继续、重试、分叉在多轮场景下仍语义清晰
- 模板、追问、参数上下文与版本来源联系更紧

### 6.6 自测清单

- 连续做两轮以上继续/重试/分叉
- 从作品页和任务页分别回流
- 检查多轮来源摘要是否还能清楚表达

### 6.7 建议子分支名

- `feature/phase4-version-production-flow`

## 7. Agent D4：专业版高频复用增强

### 7.1 任务目标

让专业版更明确服务高频复用、重跑和派生，而不是只停留在当前执行解释。

### 7.2 负责范围

- `src/features/studio-pro/*`
- `src/features/studio/*`
- `src/pages/app/StudioPage.tsx`

### 7.3 允许目标

- 强化结构字段与最终 Prompt / 执行的对应关系
- 强化参数快照与版本派生的联动
- 强化模板入口、重跑入口、派生入口的复用表达
- 让专业版更像结构化生产控制台

### 7.4 不要做的事

- 不要污染普通版任务流
- 不要重构版本链底层语义
- 不要开始做专家级实验台
- 不要新增大量只读信息块而无动作价值

### 7.5 交付标准

- 专业版复用感更强
- 高频重跑和派生路径更明确
- 结构字段与执行关系更容易理解

### 7.6 自测清单

- 在专业版中做结构复用和重跑
- 检查当前结果回到参数控制的路径
- 检查结构字段与执行摘要是否一致

### 7.7 建议子分支名

- `feature/phase4-pro-reuse`

## 8. Agent E4：模板向 Skill 入口演进

### 8.1 任务目标

让模板从结构入口继续往“带执行意图和进入策略的 Skill 入口”推进。

### 8.2 负责范围

- `src/pages/app/TemplatesPage.tsx`
- `src/features/prompt-templates/*`
- `src/pages/app/StudioPage.tsx`

### 8.3 允许目标

- 强化模板的推荐模式和推荐入口表达
- 强化模板到普通版/专业版/结果动作的进入策略
- 强化模板与版本链、追问链的关联表达
- 让模板页更像 Skill 入口集合，而不是结构资源页

### 8.4 不要做的事

- 不要做模板后台 CMS
- 不要重构整站导航
- 不要深改版本链底层逻辑
- 不要引入独立 Skill 市场系统

### 8.5 交付标准

- 模板更明显具备执行意图
- 模板与工作台模式和结果动作关系更明确
- 模板页更接近 Skill 入口集合

### 8.6 自测清单

- 打开模板页并检查推荐入口表达
- 从模板分别进入普通版/专业版
- 检查模板与结果动作、追问、版本语义是否更统一

### 8.7 建议子分支名

- `feature/phase4-template-skill-entry`

## 9. 第四阶段建议集成顺序

建议按以下顺序集成：

1. Agent A4
2. Agent B4
3. Agent C4
4. Agent D4
5. Agent E4

原因：

- 先压统一 schema
- 再做追问配置化
- 再增强版本多轮生产流
- 再补专业版高频复用
- 最后让模板进一步走向 Skill 入口

## 10. 第四阶段高冲突文件提醒

以下文件在第四阶段仍然容易冲突：

- `src/pages/app/StudioPage.tsx`
- `src/features/prompt-templates/*`
- `src/features/works/workReplay.ts`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`

处理建议：

- 先明确 Agent 所有权
- 非负责人只做最小必要接触
- 集成时优先人工检查 schema 命名、追问映射和版本来源是否冲突

## 11. 第四阶段总结

第四阶段的多 Agent 推进重点，是把第三阶段已经成立的结构点继续打通成结构链。

所以这一轮必须持续盯住两件事：

- 每个 Agent 不要只做局部体验优化
- 每个 Agent 都尽量让共享 schema 和主流程链更稳定
