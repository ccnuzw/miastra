# 图像 Skill 平台 V1 · 第五阶段多 Agent 任务说明书

## 文档说明

- 文档目标：把第五阶段 backlog 继续拆成可直接分发给多会话 / 多 Agent 的任务说明书。
- 文档定位：承接 [image-skill-platform-phase5-backlog-v1.md](./image-skill-platform-phase5-backlog-v1.md) 的执行分发文档。
- 当前状态：第五阶段多 Agent 执行说明文档。
- 更新时间：2026-05-05。

## 1. 使用方式

这份文档继续沿用前四阶段的分发方式，但目标已经切换到“系统能力稳定化和高频效率增强”。

推荐使用方式：

1. 继续以当前总分支 `feature/image-skill-platform-v1` 作为集成分支
2. 每个 Agent 从总分支切自己的第五阶段子分支
3. 每个 Agent 严格按自己的负责范围改动
4. 每个 Agent 完成后先自检，再合回总分支
5. 总分支统一做第五阶段集成验证

## 2. 第五阶段分工原则

- 先压共享契约，再增强可组合机制
- 先补高频效率，再补更强的对照与运行意图
- 按文件边界拆，但要明确共享对象边界
- 继续避免多个 Agent 同时深改同一个高冲突文件
- 第五阶段要特别避免“又加了一层抽象，但高频主流程并没有更稳或更快”

## 3. Agent 总览

| Agent | 任务名称 | 核心目标 | 优先级 | 是否阻塞别人 |
|---|---|---|---|---|
| Agent A5 | 共享契约深化 | 让模板、追问、执行、版本更接近稳定系统契约 | P0 | 部分 |
| Agent B5 | 追问机制可组合化 | 让追问更容易由字段、场景和默认策略自动组合 | P0 | 部分 |
| Agent C5 | 版本流高频效率增强 | 让多轮继续、重试、分叉在高频场景下更快更稳 | P0 | 部分 |
| Agent D5 | 专业版对照与校准增强 | 让专业版更适合比较、校准和派生 | P1 | 否 |
| Agent E5 | 模板 Skill 运行意图增强 | 让模板更明确决定追问路径、进入策略和结果动作优先级 | P1 | 部分 |

## 4. Agent A5：共享契约深化

### 4.1 任务目标

让模板、追问、执行、版本之间的共享对象更接近稳定系统契约，减少桥接层和重复映射。

### 4.2 负责范围

- `src/features/prompt-templates/*`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`
- `src/features/generation/*`
- `src/features/works/workReplay.ts`

### 4.3 允许目标

- 收口共享对象边界和字段约束
- 清理重复桥接和重复映射
- 强化结构对象在主流程和历史回流中的一致性
- 为后续持久化和服务端承接预留更稳接口

### 4.4 不要做的事

- 不要重构后端或数据库
- 不要为了“更纯”而打断现有主流程
- 不要引入大而全的新中台层
- 不要回滚第四阶段已有共享字段工作

### 4.5 交付标准

- 共享对象边界更清楚
- 桥接和重复映射明显减少
- 主流程和回流链中的结构对象更一致

### 4.6 自测清单

- 检查模板、追问、执行、版本的核心对象边界
- 验证同一结构对象在主流程和回流链中语义一致
- 确认没有新增命名分裂

### 4.7 建议子分支名

- `feature/phase5-shared-contract`

## 5. Agent B5：追问机制可组合化

### 5.1 任务目标

让轻量追问更容易由模板字段、场景和默认策略自动组合，而不是主要依赖手工定义。

### 5.2 负责范围

- `src/features/studio-home/*`
- `src/features/studio-consumer/*`
- `src/features/prompt-templates/*`
- `src/pages/app/StudioPage.tsx`

### 5.3 允许目标

- 强化字段驱动问题生成
- 抽象默认值和推荐值策略
- 强化追问结果到 Prompt、结果动作、回流状态的自动映射
- 在控制范围内扩展更多高频场景

### 5.4 不要做的事

- 不要做复杂自由对话
- 不要重构专业版控制区
- 不要引入另一套完全独立的追问系统
- 不要回退成继续堆散点硬编码

### 5.5 交付标准

- 字段与追问生成关系更直接
- 新场景接入追问的手工成本进一步下降
- 追问结果到主流程的映射更自动更稳定

### 5.6 自测清单

- 触发多个场景的追问
- 检查字段驱动组合是否生效
- 检查追问结果到输入区、生成态、结果动作、回流状态的映射

### 5.7 建议子分支名

- `feature/phase5-guided-flow-composition`

## 6. Agent C5：版本流高频效率增强

### 6.1 任务目标

让版本链在高频多轮创作中，不只是能理解，而且能更快消费、更快切换和更快继续。

### 6.2 负责范围

- `src/features/works/workReplay.ts`
- `src/pages/app/WorksPage.tsx`
- `src/pages/app/TasksPage.tsx`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`

### 6.3 允许目标

- 强化更结构化的节点摘要
- 强化高频多轮回流下的来源判断
- 强化模板、追问、参数与版本节点的显式对应关系
- 优化普通版和专业版对多轮版本流的消费效率

### 6.4 不要做的事

- 不要做复杂图谱
- 不要引入重量级多版本管理器
- 不要重构模板结构层
- 不要开始做完整 diff 面板

### 6.5 交付标准

- 多轮版本流更快理解
- 高级用户连续派生时心智更稳
- 版本节点与模板、追问、参数联系更显式

### 6.6 自测清单

- 连续进行多轮继续、重试、分叉
- 从作品页和任务页分别高频回流
- 检查来源判断和节点摘要是否更快被理解

### 6.7 建议子分支名

- `feature/phase5-version-efficiency`

## 7. Agent D5：专业版对照与校准增强

### 7.1 任务目标

让专业版更适合比较、校准和派生，而不只是复用当前已有基线。

### 7.2 负责范围

- `src/features/studio-pro/*`
- `src/features/studio/*`
- `src/pages/app/StudioPage.tsx`

### 7.3 允许目标

- 强化结构字段与 Prompt 的对照表达
- 强化当前版与来源版的对照表达
- 强化参数快照的校准和提示能力
- 强化专业版中的对照、派生和复用共存体验

### 7.4 不要做的事

- 不要污染普通版任务流
- 不要重构版本链底层语义
- 不要开始做完整实验台
- 不要新增大量只读信息而无校准价值

### 7.5 交付标准

- 专业版更适合做对照和校准
- 来源版与当前版关系更直观
- 参数快照更有比较价值

### 7.6 自测清单

- 在专业版里做一次来源版对照
- 检查结构字段与 Prompt 的对照表达
- 检查参数快照是否更有校准和比较价值

### 7.7 建议子分支名

- `feature/phase5-pro-compare`

## 8. Agent E5：模板 Skill 运行意图增强

### 8.1 任务目标

让模板更明确决定追问路径、进入方式和结果动作优先级，进一步接近真正运行中的 Skill 单元。

### 8.2 负责范围

- `src/pages/app/TemplatesPage.tsx`
- `src/features/prompt-templates/*`
- `src/pages/app/StudioPage.tsx`
- `src/features/studio-consumer/*`

### 8.3 允许目标

- 强化模板对追问路径的影响
- 强化模板对普通版 / 专业版进入策略的影响
- 强化模板对结果动作优先级和默认分支的影响
- 让模板页更像运行中的 Skill 入口集合

### 8.4 不要做的事

- 不要做 Skill 市场系统
- 不要重构整站导航
- 不要深改版本链底层逻辑
- 不要新增大量只读入口说明而无运行价值

### 8.5 交付标准

- 模板更明确决定追问路径和进入方式
- 模板对结果动作的优先级影响更清楚
- 模板页更像运行中 Skill 的入口集合

### 8.6 自测清单

- 从模板进入普通版和专业版
- 检查模板是否更清楚影响追问路径和默认动作
- 检查模板与结果动作优先级是否更一致

### 8.7 建议子分支名

- `feature/phase5-template-skill-runtime`

## 9. 第五阶段建议集成顺序

建议按以下顺序集成：

1. Agent A5
2. Agent B5
3. Agent C5
4. Agent D5
5. Agent E5

原因：

- 先把共享契约继续压实
- 再让追问走向字段驱动组合
- 再增强高频版本流效率
- 再补专业版对照与校准
- 最后让模板进一步承接 Skill 运行意图

## 10. 第五阶段高冲突文件提醒

以下文件在第五阶段仍然容易冲突：

- `src/pages/app/StudioPage.tsx`
- `src/features/prompt-templates/*`
- `src/features/works/workReplay.ts`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`

处理建议：

- 先明确 Agent 所有权
- 非负责人只做最小必要接触
- 集成时优先人工检查共享对象边界、追问组合逻辑和版本节点摘要是否冲突

## 11. 第五阶段总结

第五阶段的多 Agent 推进重点，是把第四阶段已经较统一的结构能力继续推进成更稳定、更高频可用的系统能力。

所以这一轮必须持续盯住两件事：

- 每个 Agent 不要只补更多配置项
- 每个 Agent 都尽量让共享契约和高频主流程更稳、更快
