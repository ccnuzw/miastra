# 图像 Skill 平台 V1 · 第九阶段多 Agent 任务说明书

## 文档说明

- 文档目标：把第九阶段 backlog 继续拆成可直接分发给多会话 / 多 Agent 的任务说明书。
- 文档定位：承接 [image-skill-platform-phase9-backlog-v1.md](./image-skill-platform-phase9-backlog-v1.md) 的执行分发文档。
- 当前状态：第九阶段多 Agent 执行说明文档。
- 更新时间：2026-05-05。

## 1. 使用方式

这份文档继续沿用前八阶段的分发方式，但目标已经切换到“最终一致性、最终收边、最终回归与上线执行准备”。

推荐使用方式：

1. 继续以当前总分支 `feature/image-skill-platform-v1` 作为集成分支
2. 每个 Agent 从总分支切自己的第九阶段子分支
3. 每个 Agent 严格按自己的负责范围改动
4. 每个 Agent 完成后先自检，再合回总分支
5. 总分支统一做第九阶段集成验证

## 2. 第九阶段分工原则

- 先做主干最终一致性，再做主链和专业版最终收边
- 先统一异常态和恢复态，再补最终回归与上线执行准备
- 按文件边界拆，但要明确 contract、runtime、版本流和专业版连续链的最终共享边界
- 继续避免多个 Agent 同时深改同一个高冲突文件
- 第九阶段要特别避免“再做一轮新功能”，这一轮只做最终收口和上线执行准备

## 3. Agent 总览

| Agent | 任务名称 | 核心目标 | 优先级 | 是否阻塞别人 |
|---|---|---|---|---|
| Agent A9 | Contract / Runtime 最终一致性收口 | 完成主干结构最终一致性检查与封口 | P0 | 部分 |
| Agent B9 | 高频主链最终收边 | 完成普通版、版本流、模板入口的最终主链收边 | P0 | 部分 |
| Agent C9 | 专业版连续链最终压实 | 完成专业版连续决策与连续派生的最终压实 | P0 | 部分 |
| Agent D9 | 异常态与恢复态最终统一 | 完成双模式关键边界态最终统一 | P1 | 否 |
| Agent E9 | 最终回归与上线执行准备 | 完成手工验收、回归视角和上线执行准备 | P1 | 部分 |

## 4. Agent A9：Contract / Runtime 最终一致性收口

### 4.1 任务目标

完成主干结构最终一致性检查与封口。

### 4.2 负责范围

- `src/features/generation/*`
- `src/features/prompt-templates/*`
- `src/features/studio-consumer/*`
- `src/features/works/workReplay.ts`

### 4.3 允许目标

- 做 contract / runtime 最终一致性检查
- 清理残余低频 bridge / fallback / runtime 特例
- 明确最终主干判断边界
- 压缩最后一批游离在主干外的结构语义

### 4.4 不要做的事

- 不要重构服务端或数据库
- 不要重开新的结构层
- 不要扩新功能面
- 不要回滚第八阶段已有最终封口工作

### 4.5 交付标准

- contract / runtime 一致性更明确
- 残余特例继续减少
- 主干判断更清楚

### 4.6 自测清单

- 检查主流程、回流、版本恢复、入口恢复是否共享同一主干判断
- 检查低频 bridge / fallback / runtime 特例是否继续减少
- 确认没有新增平行主干

### 4.7 建议子分支名

- `feature/phase9-contract-runtime-final`

## 5. Agent B9：高频主链最终收边

### 5.1 任务目标

完成普通版、版本流、模板入口的最终主链收边。

### 5.2 负责范围

- `src/features/studio-consumer/*`
- `src/pages/app/StudioPage.tsx`
- `src/pages/app/WorksPage.tsx`
- `src/pages/app/TasksPage.tsx`
- `src/pages/app/TemplatesPage.tsx`

### 5.3 允许目标

- 再压普通版高频主链
- 再压版本流高频回流路径
- 再压模板入口到工作台的最终体验链
- 清理仍然存在的高频主链理解拐点

### 5.4 不要做的事

- 不要做大改版 UI
- 不要重构 contract / runtime 主干
- 不要扩新入口
- 不要为了“更完整”而增加更多非必要说明

### 5.5 交付标准

- 高频主链最终更顺
- 回流路径最终更短
- 模板入口最终更自然

### 5.6 自测清单

- 从首页、模板入口、结果动作、作品回流、任务回流分别进入工作台
- 检查普通版高频主链是否更短
- 检查模板入口到工作台链路是否更自然

### 5.7 建议子分支名

- `feature/phase9-mainflow-polish`

## 6. Agent C9：专业版连续链最终压实

### 6.1 任务目标

完成专业版连续决策与连续派生的最终压实。

### 6.2 负责范围

- `src/features/studio-pro/*`
- `src/features/studio/*`
- `src/pages/app/StudioPage.tsx`

### 6.3 允许目标

- 再压专业版连续派生链
- 再压专业版重跑 / 对照 / 校准链
- 清理专业版剩余使用跳跃点
- 让连续判断更像最终生产流

### 6.4 不要做的事

- 不要污染普通版任务流
- 不要重构版本链底层语义
- 不要开始做完整实验台
- 不要增加大量只读信息而无连续判断价值

### 6.5 交付标准

- 专业版连续链最终更稳
- 跳跃点更少
- 连续判断更自然

### 6.6 自测清单

- 在专业版里做一次连续派生和一次连续重跑
- 检查对照、校准、执行建议之间是否更顺
- 检查剩余跳跃点是否减少

### 6.7 建议子分支名

- `feature/phase9-pro-final-pass`

## 7. Agent D9：异常态与恢复态最终统一

### 7.1 任务目标

完成双模式关键边界态最终统一。

### 7.2 负责范围

- `src/pages/app/StudioPage.tsx`
- `src/pages/app/TemplatesPage.tsx`
- `src/pages/app/WorksPage.tsx`
- `src/pages/app/TasksPage.tsx`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`

### 7.3 允许目标

- 补最终异常态、恢复态、降级态
- 统一双模式边界反馈
- 统一模板入口和回流链的恢复反馈
- 清理剩余边界态不一致问题

### 7.4 不要做的事

- 不要扩新大型功能模块
- 不要重构整站导航
- 不要深改 contract 和 runtime 主干
- 不要只做视觉修饰而不提升边界态一致性

### 7.5 交付标准

- 边界态更统一
- 恢复态更自然
- 降级态更清楚

### 7.6 自测清单

- 检查普通版、专业版、模板入口、回流链的异常态与恢复态
- 检查关键降级态反馈
- 检查不同入口下边界态文案是否一致

### 7.7 建议子分支名

- `feature/phase9-state-unification`

## 8. Agent E9：最终回归与上线执行准备

### 8.1 任务目标

完成手工验收、回归视角和上线执行准备。

### 8.2 负责范围

- `docs/*`
- 必要时少量涉及 `src/pages/app/*` 的状态文案和反馈

### 8.3 允许目标

- 整理最终手工验收视角
- 整理最终回归重点
- 整理上线执行准备相关文档或提示
- 补充极少量与回归执行直接相关的状态文案

### 8.4 不要做的事

- 不要再扩新业务功能
- 不要重构页面结构
- 不要深改主流程代码
- 不要把回归文档写成新一轮 PRD

### 8.5 交付标准

- 最终回归重点更清楚
- 手工验收更容易执行
- 上线执行准备更完整

### 8.6 自测清单

- 检查回归清单是否覆盖双模式、模板入口、版本回流、专业版连续链
- 检查验收步骤是否可直接执行
- 检查上线执行准备文档是否足够清楚

### 8.7 建议子分支名

- `feature/phase9-release-execution`

## 9. 第九阶段建议集成顺序

建议按以下顺序集成：

1. Agent A9
2. Agent B9
3. Agent C9
4. Agent D9
5. Agent E9

原因：

- 先做主干最终一致性收口
- 再压高频主链
- 再压专业版连续链
- 然后统一异常态与恢复态
- 最后整理最终回归与上线执行准备

## 10. 第九阶段高冲突文件提醒

以下文件在第九阶段仍然容易冲突：

- `src/pages/app/StudioPage.tsx`
- `src/features/prompt-templates/*`
- `src/features/works/workReplay.ts`
- `src/features/studio-consumer/*`
- `src/features/studio-pro/*`
- `src/features/generation/*`

处理建议：

- 先明确 Agent 所有权
- 非负责人只做最小必要接触
- 集成时优先人工检查主干一致性、高频主链、专业版连续链和边界态统一是否互相覆盖

## 11. 第九阶段总结

第九阶段的多 Agent 推进重点，是把前八阶段已经非常接近上线准备状态的系统，最终压成可以进入最终回归、最终验收和上线执行的状态。

所以这一轮必须持续盯住两件事：

- 每个 Agent 不要再打开新的能力层
- 每个 Agent 都尽量让最终一致性、最终收边和上线执行准备更近一步
