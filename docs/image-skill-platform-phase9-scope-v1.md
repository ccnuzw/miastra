# 图像 Skill 平台 V1 · 第九阶段开发目标与范围

## 文档说明

- 文档目标：在第八阶段复盘基础上，正式定义第九阶段的开发目标、范围边界、优先级和不做项。
- 文档定位：承接 [image-skill-platform-phase8-review-v1.md](./image-skill-platform-phase8-review-v1.md) 的第九阶段范围文档。
- 当前状态：第九阶段范围主文档。
- 更新时间：2026-05-05。

## 1. 第九阶段结论先行

第九阶段不再解决“V1 主结构是否稳定”的问题，而是开始解决：

- contract 与 runtime 能不能完成最终一致性收口
- 高频主链能不能完成最后一轮行为统一
- 专业版能不能完成最终连续使用链压实
- 双模式异常态和恢复态能不能完成最终统一
- 整个工作台能不能进入最终验收与上线执行准备

一句话总结：

第八阶段做最终封口准备，第九阶段开始做最终验收与上线执行准备。

## 2. 为什么现在进入第九阶段

经过前八阶段之后，当前已经具备以下前提：

- contract 已非常接近唯一事实来源
- runtime 例外已经明显减少
- 版本流已非常接近高频生产链
- 专业版已接近连续决策终态
- 双模式、模板入口、回流链已经进入明显回归准备区

因此，当前最合理的推进方向，不是再加能力，而是做最终一致性检查、最终主链收边和最终上线执行准备。

## 3. 第九阶段总目标

第九阶段总目标可以收口成 3 句：

1. 完成 V1 主干结构的一致性最终收口
2. 完成高频主链和专业版连续链的最终使用收边
3. 完成上线前回归、验收与执行准备

## 4. 第九阶段产品原则

- 优先最终一致性，不优先继续优化外围体验
- 优先最终验收，不优先继续扩展功能面
- 优先回归、异常态、恢复态闭合，不优先新入口
- 优先上线执行准备，不优先远期平台能力

## 5. 第九阶段范围总览

建议把第九阶段收口成 5 个方向。

| 方向 | 名称 | 目标 | 优先级 |
|---|---|---|---|
| P91 | Contract / Runtime 最终一致性收口 | 完成主干结构最终一致性检查与封口 | P0 |
| P92 | 高频主链最终收边 | 完成普通版、版本流、模板入口的最终主链收边 | P0 |
| P93 | 专业版连续链最终压实 | 完成专业版连续决策与连续派生的最终压实 | P0 |
| P94 | 异常态与恢复态最终统一 | 完成双模式关键边界态最终统一 | P1 |
| P95 | 最终回归与上线执行准备 | 完成手工验收、回归视角和上线执行准备 | P1 |

## 6. 第九阶段最值得优先做的 6 个任务

1. 做 contract / runtime 最终一致性检查与封口
2. 清理剩余低频主链行为差异
3. 再压一次版本流和模板入口的高频主链
4. 再压一次专业版连续决策链
5. 补齐最终异常态、恢复态和降级态统一
6. 整理最终回归、验收和上线执行准备

## 7. 第九阶段明确不做

- 不做新大型功能模块
- 不做团队协作
- 不做项目空间
- 不做开放 API 平台
- 不做模板 CMS
- 不做 Skill 市场
- 不做企业能力扩张

## 8. 第九阶段成功标准

- contract / runtime 最终一致性更清楚
- 高频主链最终更顺
- 专业版连续链最终更稳
- 异常态与恢复态最终更统一
- 系统进入最终回归与上线执行准备状态

## 9. 第九阶段总结

第九阶段不是继续做产品扩张，而是做 V1 最终验收与上线执行准备。

一句话收口：

第八阶段让系统接近可以进入上线准备，第九阶段要让它真正进入上线执行准备。
