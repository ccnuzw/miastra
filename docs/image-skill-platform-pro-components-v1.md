# 图像 Skill 平台 V1 · 专业版组件清单与状态清单

## 文档说明

- 文档目标：把专业版创作页和结果页继续拆成组件清单与状态清单，作为后续原型和开发实现依据。
- 文档定位：承接 [image-skill-platform-pro-wireframes-v1.md](./image-skill-platform-pro-wireframes-v1.md) 的组件级文档。
- 当前状态：专业版组件与状态文档。
- 更新时间：2026-05-05。

## 1. 文档范围

本文件优先覆盖专业版两张核心页面：

1. 专业版创作页
2. 专业版结果页

拆解维度包括：

- 页面组件清单
- 组件状态清单

- 页面级异常和空态

## 2. 专业版创作页组件清单

### 2.1 页面目标

- 让用户高控制地发起生成或编辑任务
- 让模板、Prompt、参数和执行策略都可见且可控

### 2.2 组件清单

专业版创作页建议包含以下组件：

1. 顶部全局操作条

4. Prompt 输入与查看组件
5. 图片输入组件
6. 参数分组组件
7. 执行设置组件

## 4. 页面级异常与空态建议

### 4.1 专业版创作页

- 无模板空态：显示推荐模板
- Prompt 为空态：提示先输入原始需求
- 图片上传失败态：允许重新上传
- 执行失败态：保留所有配置并支持切换执行策略

### 4.2 专业版结果页

- 无候选隐藏态：只显示当前主结果
- 图片加载失败态：允许重载
- 版本链为空态：显示当前结果为链路起点
- 导出失败态：支持重试

## 5. 状态共享要求

专业版创作页与结果页必须共享以下状态：

- 当前任务输入
- 当前 Prompt
- 当前模板
- 当前图片输入
- 当前参数快照
- 当前 Provider / model
- 当前结果与候选结果
- 当前版本链关系

要求：

- 从创作页到结果页不丢任何上下文
- 从结果页回创作页不丢结果链路
- 从普通版切到专业版时也要保留以上状态

## 6. 文档结论

专业版组件清单与状态清单的核心目标，是把“更多控制能力”拆成可以实现、可以维护、可以扩展的页面单元。

这样后续继续往交互流、文案和布局决策推进时，就不会把专业版重新做成一块混乱的大面板。
