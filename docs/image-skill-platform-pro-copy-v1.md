# 图像 Skill 平台 V1 · 专业版文案与按钮文案清单

## 文档说明

- 文档目标：统一专业版创作页、结果页及相关高级视图中的标题文案、说明文案、按钮文案、状态文案与失败提示文案。
- 文档定位：承接 [image-skill-platform-pro-interactions-v1.md](./image-skill-platform-pro-interactions-v1.md) 的原型级文案文档。
- 当前状态：专业版文案规范文档。
- 更新时间：2026-05-05。

## 1. 文档原则

专业版文案的目标不是“更技术化”，而是“更专业但仍然可理解”。

需要遵循以下原则：

- 允许出现专业概念，但必须清楚
- 文案要强调控制感、复用感和透明度
- 不做过度包装，不假装简单
- 不用晦涩缩写作为主文案
- 失败提示必须告诉用户还能怎么做

一句话原则：

专业版文案要让用户感觉“我可以掌控它”，而不是“我被一堆技术词淹没了”。

## 2. 全局文案规范

### 2.1 专业版允许直接使用的表达

- 模板
- Prompt
- 参数
- 版本链
- Provider
- 模型
- 变体
- 重跑
- 快照
- 执行记录

### 2.2 仍然不建议过度使用的表达

- 原始渲染管线
- 模板 DSL
- Runtime Mode
- 结构化生成图谱
- 深度参数编排

这些内容可以存在于说明文档或详情中，但不应成为页面主按钮语言。

### 2.3 语气要求

- 简洁
- 准确
- 不花哨
- 不夸张
- 偏工作台语气，不偏助手撒娇语气

### 2.4 按钮命名规则

- 主按钮优先直接表达动作
- 尽量避免模糊词
- 同一页按钮不能语义重叠

## 7. 不建议使用的文案

以下表达不建议直接用于专业版主界面：

- 启动运行时
- 执行图像管线
- 渲染模板 DSL
- 调用底层模型路由器
- 写入结构化节点

原因：

- 虽然专业版允许更多技术透明度，但仍然应该是产品语言，不是内部工程语言

## 8. 文档结论

专业版文案的核心，不是让页面显得更“高级”，而是让专业用户在使用时有三种感受：

- 我知道系统做了什么
- 我知道我能控制什么
- 我知道我失败后还能怎么继续

只要做到这三点，专业版文案就达到了它的目标。
