# 图像 Skill 平台 V1 · 专业版交互流与状态流

## 文档说明

- 文档目标：定义专业版创作页和结果页的关键交互流、系统响应和状态流转。
- 文档定位：承接 [image-skill-platform-pro-components-v1.md](./image-skill-platform-pro-components-v1.md) 的交互级文档。
- 当前状态：专业版交互与状态文档。
- 更新时间：2026-05-05。

## 1. 文档范围

本文件优先覆盖专业版核心主路径：

1. 专业版创作页发起任务
2. 模板与 Prompt 干预
3. 参数与执行设置修改
4. 生成与编辑执行
5. 专业版结果分析
6. 参数重跑与继续修改
7. 版本链回流
8. 普通版与专业版切换保持上下文

关注重点：

- 用户做了什么
- 系统做了什么
- 页面状态如何变化

## 2. 交互总原则

专业版交互必须满足以下要求：

- 控制力增强，但不能打断主结果判断
- 系统能力透明，但不能让页面失去层次
- 用户的每次干预都应可追溯
- 参数、Prompt、模板和结果应形成闭环
- 切换模式、切换结果、重跑任务时都不能丢失上下文

## 3. 专业版创作页交互流

## 3.1 从普通版切到专业版

### 用户目标

- 在当前任务基础上获得更高控制力

### 触发入口

- 创作页右上角：`进入进阶模式`

### 交互步骤

1. 用户在普通版创作页点击 `进入进阶模式`。
2. 系统保留当前任务输入、图片、偏好、候选结果和最近一次上下文。
3. 页面切换为专业版创作页视图。
4. 显示模板、Prompt、参数分组、Provider / model 等专业区块。

### 页面变化

- 页面从任务导向视图切换为控制导向视图
- 当前任务标题不变
- 当前结果与图片输入不丢失

### 成功标准

- 切换后用户不需要重新填写任何内容

## 3.2 模板选择与变体切换流

### 用户目标

- 通过模板和变体更精确控制生成方向

### 触发入口

- 模板选择组件
- 模板变体切换组件

### 交互步骤

1. 用户打开模板区。
2. 用户选择某个模板。
3. 系统更新当前任务所绑定的模板上下文。
4. 用户继续选择模板变体。
5. 系统刷新 Prompt 补全策略和参数推荐值。

### 系统动作

- 记录模板 ID 与模板版本
- 刷新模板说明
- 刷新默认参数
- 更新 Prompt 预览

### 页面变化

- 当前模板卡更新
- Prompt 区与参数区内容变化
- 右侧任务快照中的模板信息同步更新

### 异常处理

- 切换模板时不应清空原始需求输入
- 若新模板与旧图片输入不兼容，应给出提示而不是直接清空

## 3.3 Prompt 查看与编辑流

### 用户目标

- 查看系统最终 Prompt
- 手动调整 Prompt 再执行

### 触发入口

- Prompt 区
- 复制 Prompt
- 编辑 Prompt

### 交互步骤

1. 用户查看当前 Prompt 摘要。
2. 用户点击展开全文。
3. 用户可复制 Prompt，或进入编辑状态。
4. 用户编辑完成后保存。
5. 页面进入“Prompt 已修改待重跑”状态。

### 系统动作

- 保留系统原始补全版本
- 记录当前用户编辑版本
- 标记本次任务为“用户干预 Prompt”

### 页面变化

- Prompt 区从摘要态切换到展开态
- 编辑后显示“已修改”标记
- 主操作区出现“基于当前 Prompt 重跑”能力

### 异常处理

- 用户取消编辑时，恢复最近保存版本
- 不清空系统默认 Prompt

## 3.4 图片输入流

### 用户目标

- 更精确地控制原图、参考图和蒙版的使用

### 交互步骤

1. 用户上传原图、参考图或蒙版。
2. 系统校验类型和数量。
3. 图片缩略卡显示在输入区。
4. 页面根据任务类型更新可用参数和 Prompt 策略。

### 系统动作

- 识别图片用途
- 记录图片与任务关系
- 更新编辑任务的上下文

### 页面变化

- 图片输入区从空态切换为已上传态
- 对应编辑参数解锁或更新

### 异常处理

- 上传失败时不影响当前任务其他信息
- 支持单张重试

## 3.5 参数修改流

### 用户目标

- 通过高级参数控制输出结果

### 参数分组

- 构图与比例
- 风格与强度
- 输出与质量
- 高级控制

### 交互步骤

1. 用户修改某一组参数。
2. 系统立即记录参数变化。
3. 参数快照区同步更新。
4. 若某些改动影响 Prompt，可刷新 Prompt 预览。

### 页面变化

- 当前修改的分组进入“已修改”状态
- 参数快照显示变化
- 页面主按钮仍保持可执行

### 异常处理

- 参数恢复默认应支持单组重置
- 修改参数不应导致其他组被误清空

## 3.6 Provider / model 切换流

### 用户目标

- 选择执行路径和模型策略

### 触发入口

- 执行设置区

### 交互步骤

1. 用户打开 Provider / model 设置。
2. 用户切换 Provider 或 model。
3. 系统更新当前执行配置。
4. 右侧任务快照同步刷新。

### 系统动作

- 记录当前执行配置
- 标记本次任务使用的 Provider 与 model

### 页面变化

- 顶部 Provider 状态更新
- 执行设置区显示当前值

### 异常处理

- 若 Provider 不可用，提示可切回平台默认模式
- 不因执行设置错误清空当前任务

## 3.7 发起执行流

### 用户目标

- 用当前模板、Prompt、参数和 Provider 发起一次可追溯任务

### 触发入口

- 生成
- 生成 4 张
- 基于当前重跑

### 交互步骤

1. 用户点击主操作按钮。
2. 系统校验必要条件。
3. 系统保存当前 Prompt 快照、参数快照、图片输入快照和执行设置。
4. 页面进入执行中态。
5. 中间结果区显示过程状态。
6. 执行完成后生成结果并进入结果页。

### 系统动作

- 固化任务快照
- 创建任务与任务运行记录
- 建立当前结果与父节点关系

### 页面变化

- 主按钮进入禁用态
- 生成过程状态组件显示阶段
- 执行完成后切到结果页或结果态

### 异常处理

- 失败时保留所有快照
- 用户可直接调整参数或切换 Provider 后重试

## 4. 专业版结果页交互流

## 4.1 首次结果分析流

### 用户目标

- 看结果是否符合预期
- 判断是否继续调参或继续改

### 交互步骤

1. 结果页展示主结果。
2. 用户查看大图。
3. 用户切换候选结果对比。
4. 同时查看 Prompt、参数和模板信息。

### 页面变化

- 主结果区优先可见
- Prompt 快照与参数快照区同步当前选中结果

## 4.2 候选结果切换流

### 用户目标

- 快速比较不同候选图

### 交互步骤

1. 用户点击某个候选缩略图。
2. 主结果切换。
3. 结果说明区更新。
4. Prompt / 参数 / 执行信息跟随当前结果切换。

### 成功标准

- 用户看到的不只是图切了，相关上下文也切了

## 4.3 参数重跑流

### 用户目标

- 用当前结果对应的参数快速再跑一轮

### 触发入口

- `基于当前参数重跑`
- `复制参数重跑`

### 交互步骤

1. 用户在结果页点击参数重跑。
2. 系统读取当前结果对应的参数与 Prompt 快照。
3. 页面回到专业版创作页或直接发起重跑。
4. 新任务建立为当前结果的子节点。

### 页面变化

- 若需修改，回到创作页并带全量上下文
- 若直接执行，结果页进入执行中态

### 系统动作

- 保存“来源结果节点”
- 创建新任务子链路

## 4.4 继续高精度修改流

### 用户目标

- 在当前结果基础上继续深调，而不是回到初始任务

### 主要动作

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景

### 通用交互步骤

1. 用户点击某个继续修改动作。
2. 系统读取当前结果、Prompt 和参数快照。
3. 页面回到专业版创作页，并自动带入对应修改上下文。

### 页面变化

- 当前结果成为新的基础图
- 相关模板与参数可自动聚焦到对应区域

### 设计要求

- 专业版继续修改不能只给一个动作，还要保留上下文透明度

## 4.5 版本链回流流

### 用户目标

- 从任意历史节点继续创作

### 触发入口

- 版本链组件中的节点

### 交互步骤

1. 用户点击版本链中的某个节点。
2. 系统切换当前上下文到该节点。
3. 用户可以：
   - 查看它的结果
   - 查看它的 Prompt / 参数
   - 从这里继续创作

### 系统动作

- 加载节点上下文
- 更新当前节点标记

### 页面变化

- 当前结果区切换到目标节点
- Prompt 与参数快照同步切换

## 4.6 保存与导出流

### 用户目标

- 把当前结果和当前方法沉淀下来

### 主要动作

- 保存到作品
- 下载结果图
- 导出 Prompt
- 导出参数快照

### 通用交互步骤

1. 用户点击导出或保存动作。
2. 系统执行保存或导出。
3. 页面给出轻量成功反馈。

### 页面变化

- 顶部或按钮旁显示成功状态
- 不打断当前结果分析

## 5. 专业版与普通版切换流

## 5.1 普通版 -> 专业版

必须保留：

- 当前任务输入
- 已上传图片
- 当前模板上下文
- 当前结果或候选结果
- 当前偏好与参数

进入专业版后新增显示：

- 模板区
- Prompt 区
- 参数分组
- Provider / model 区
- 快照与版本链

## 5.2 专业版 -> 普通版

必须保留：

- 当前任务
- 当前结果
- 当前图片输入

隐藏内容：

- Prompt 细节
- Provider 控制
- 高级参数
- 版本细节

## 6. 页面级状态流总表

### 6.1 专业版创作页状态

- 初始空白态
- 已配置待执行态
- Prompt 编辑中态
- 执行中态
- 结果可用态
- 执行失败态
- 历史结果回流态

### 6.2 专业版结果页状态

- 首次结果展示态
- 多候选比较态
- 参数重跑准备态
- 继续修改准备态
- 保存成功态
- 导出成功态

## 7. 失败与恢复策略

### 7.1 创作页

- 执行失败后，Prompt、参数、图片输入全部保留
- Provider 错误时，允许切回默认 Provider
- Prompt 编辑失败时，恢复到最近保存版本

### 7.2 结果页

- 某个结果节点加载失败时，不影响其他节点浏览
- 导出失败时，允许单独重试
- 继续修改失败时，不丢当前结果上下文

## 8. 文档结论

专业版交互流的核心不是让用户“看到很多能力”，而是让用户在每一步都能：

- 明确当前控制了什么
- 明确系统执行了什么
- 明确结果来自哪里
- 明确下一步如何继续

只要这 4 点成立，专业版就不只是更复杂，而是更有生产价值的工作台。
