# 图像 Skill 平台 V1 · 专业版页面结构文档

## 文档说明

- 文档目标：定义专业版的核心页面范围、页面职责、页面之间的关系与每个页面的关键承载内容。
- 文档定位：承接 [image-skill-platform-pro-v1-prd.md](./image-skill-platform-pro-v1-prd.md) 的页面结构文档。
- 当前状态：专业版页面规划文档。
- 更新时间：2026-05-05。

## 1. 文档范围

本文件聚焦专业版 5 个核心页面或视图：

1. 专业版创作页
2. 专业版结果页
3. 专业版历史与版本页
4. 专业版模板视图
5. 专业版 Provider / 执行设置视图

本文件不进入线框级细节，重点回答：

- 专业版有哪些核心页面
- 每个页面解决什么问题
- 每个页面的主要信息结构是什么
- 页面之间如何协同

## 2. 专业版页面总体原则

专业版页面设计应遵循以下原则：

- 页面按控制目标组织，而不是按技术模块组织
- 每张页面都要围绕“可控、可复用、可追溯”展开
- 页面之间共享任务、结果、参数和版本上下文
- 不把专业版做成“参数堆砌站”
- 高级信息可见，但必须分区清楚

一句话：

专业版页面不是把所有能力摊开，而是让用户在正确的位置看到正确的控制能力。

## 3. 页面一：专业版创作页

### 3.1 页面定位

专业版创作页是专业用户的主工作台。

它解决的问题是：

- 我要更精确地发起一次生成或编辑任务
- 我要控制 Prompt、模板、参数、图片输入和执行方式

### 3.2 核心职责

- 选择任务类型
- 选择或切换模板
- 查看与编辑 Prompt
- 调整高级参数
- 上传原图 / 参考图 / 蒙版图
- 选择 Provider / model
- 发起生成或编辑
- 预览生成结果

### 3.3 关键承载内容

- 模板选择区
- Prompt 区
- 参数分组区
- 图片输入区
- Provider / model 区
- 结果预览区

### 3.4 与普通版区别

普通版创作页重点是“帮用户顺手开始”。  
专业版创作页重点是“给用户更多控制力”。

### 3.5 专业版创作页在体系中的优先级

这是专业版最重要的页面。

## 4. 页面二：专业版结果页

### 4.1 页面定位

专业版结果页不是单纯的查看页，而是结果分析、复用和继续创作中心。

它解决的问题是：

- 我想对比不同候选图
- 我想看这张图是怎么来的
- 我想在当前结果基础上继续调
- 我想复制 Prompt 或参数重跑

### 4.2 核心职责

- 主结果与候选结果展示
- Prompt 与参数快照查看
- 模板和 Provider 信息查看
- 继续修改与重跑
- 结果对比与版本追踪

### 4.3 关键承载内容

- 结果主展示区
- 候选结果对比区
- Prompt 快照区
- 参数快照区
- 继续修改动作区
- 版本链区

### 4.4 与普通版区别

普通版结果页强调“继续改很顺手”。  
专业版结果页强调“结果可分析、可复用、可追溯”。

### 4.5 专业版结果页在体系中的优先级

这是专业版第二重要的页面。

## 5. 页面三：专业版历史与版本页

### 5.1 页面定位

专业版历史页不是简单任务列表，而是历史任务、结果资产和版本链的管理页。

它解决的问题是：

- 我想快速找回某次成功结果
- 我想看某条结果链是怎么演变的
- 我想筛选不同 Skill、模板、Provider 下的任务
- 我想从历史节点直接继续创作

### 5.2 核心职责

- 任务列表与筛选
- 结果列表与筛选
- 版本链浏览
- 参数复用
- 从历史继续创作

### 5.3 关键承载内容

- 历史任务列表
- 结果缩略图区
- 多条件筛选区
- 版本关系视图区
- 快速重跑 / 继续编辑入口

### 5.4 与普通版区别

普通版历史页重在“找回和继续”。  
专业版历史页重在“管理、筛选、复用、追溯”。

## 6. 页面四：专业版模板视图

### 6.1 页面定位

模板视图是专业版中“能力复用”的核心入口。

它解决的问题是：

- 我想选更合适的模板
- 我想理解模板结构
- 我想切模板变体
- 我想保存常用模板

### 6.2 核心职责

- 模板分类浏览
- 模板结构查看
- 模板变体切换
- 模板收藏与常用
- 从模板进入创作页

### 6.3 关键承载内容

- 模板分类列表
- 模板卡片列表
- 模板详情区
- 模板变体区
- 模板示例结果区

### 6.4 专业版模板视图的意义

普通版中模板更多是隐式能力。  
专业版中模板应成为显式资产。

## 7. 页面五：专业版 Provider / 执行设置视图

### 7.1 页面定位

这是专业版中负责执行透明度和执行控制的页面或侧边视图。

它解决的问题是：

- 我要知道当前任务走哪个 Provider
- 我要切换模型或执行策略
- 我要管理平台模式和自带模式
- 我要在失败时快速切换执行路径

### 7.2 核心职责

- Provider 选择
- model 选择
- 默认执行模式查看
- 执行失败切换
- 基础成本与结果偏好设置

### 7.3 关键承载内容

- Provider 列表
- model 列表
- 当前默认设置
- 失败时切换策略
- 保存设置按钮

### 7.4 是否必须独立成页

V1 不一定必须是完整独立页，也可以是：

- 页面内高级侧栏
- 设置页子视图
- 创作页右侧扩展面板

但无论以什么形式存在，这部分能力都必须被清楚定义。

## 8. 专业版页面之间的关系

专业版页面应形成以下主路径：

### 路径一：从模板开始

- 模板视图
- 专业版创作页
- 专业版结果页

### 路径二：从历史复用开始

- 历史与版本页
- 专业版创作页
- 专业版结果页

### 路径三：从当前结果继续深调

- 专业版结果页
- 专业版创作页
- 专业版结果页

### 路径四：执行策略微调

- 专业版创作页
- Provider / 执行设置视图
- 专业版创作页

## 9. 页面优先级建议

为了控制 V1 范围，建议按以下优先级推进：

### P0 页面

- 专业版创作页
- 专业版结果页

### P1 页面

- 专业版历史与版本页
- 专业版模板视图

### P2 页面 / 视图

- Provider / 执行设置视图

说明：

- 即使 Provider 能力要做，UI 不一定要先做成复杂独立页
- 创作页和结果页应先把主链路跑通

## 10. 与普通版的页面协同关系

### 10.1 首页

普通版首页仍然是默认入口。  
专业用户通常从普通版首页进入，再切到专业版创作页。

### 10.2 创作页

普通版与专业版最重要的协同页。

两者应共享：

- 当前任务
- 当前图片
- 当前结果
- 当前历史上下文

### 10.3 结果页

普通版与专业版都必须支持“继续修改”，但专业版要额外承载：

- Prompt
- 参数
- 版本
- Provider 信息

### 10.4 历史页

普通版用于“找回”。  
专业版用于“管理和复用”。

## 11. V1 页面范围建议

如果控制开发节奏，专业版第一阶段建议只保证以下页面成型：

- 专业版创作页
- 专业版结果页

并为以下页面预留结构：

- 专业版历史与版本页
- 专业版模板视图
- Provider / 执行设置视图

## 12. 文档结论

专业版页面体系的核心不是“多几个高级页”，而是把专业用户最在意的 5 类能力放在正确页面中承载：

- 创建与控制：创作页
- 分析与继续：结果页
- 追溯与复用：历史与版本页
- 模板资产：模板视图
- 执行策略：Provider / 执行设置视图

只要这 5 个页面角色清楚，后续再进入线框和交互时就不会散。
