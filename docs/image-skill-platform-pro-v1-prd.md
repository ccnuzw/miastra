# 图像 Skill 平台 V1 · 专业版 PRD

## 文档说明

- 文档目标：定义图像 Skill 平台专业版的产品定位、目标用户、能力边界和一期交付范围。
- 文档定位：承接 [image-skill-platform-v1-prd.md](./image-skill-platform-v1-prd.md) 的专业版主文档。
- 当前状态：专业版产品规划主文档。
- 更新时间：2026-05-05。

## 1. 产品定义

### 1.1 一句话定义

专业版是一套面向高频创作者与专业使用者的控制导向工作台，让用户在图片生成、编辑、复用和批量生产过程中获得更高的透明度、可控性和复用效率。

### 1.2 专业版的角色

如果普通用户版解决的是：

- 快速开始
- 少输入
- 少决策
- 顺手继续改

那么专业版解决的是：

- 更精确控制结果
- 减少重复试错
- 复用成功经验
- 管理 Prompt、模板、参数和版本
- 把单次出图升级为稳定生产

### 1.3 专业版不是

- 普通用户版多几个高级开关
- 一块把所有参数都摊开的控制面板
- 直接暴露底层 API 的技术调试页
- 只适合开发者的内部工具

### 1.4 专业版应该是

- 有结构的高级工作台
- 参数更多，但分组清晰
- 可看见系统在做什么
- 可在成功经验上继续迭代
- 可把创作转成可复用生产流程

## 2. 背景与必要性

即使产品主要面向普通用户，专业版也不能缺少。

原因：

- 高级用户需要稳定复用与更高控制力
- 内容运营与电商用户需要生产效率
- 部分用户需要看 Prompt、模板和版本链
- 部分用户需要切换 Provider、模型和执行策略
- 如果没有专业版，普通版最终会被迫承担过多复杂能力，反而损害普通用户体验

因此正确做法不是只做普通版，而是：

- 用普通版承接大多数用户
- 用专业版承接高控制、高复用、高频生产需求

## 3. 目标用户

### 3.1 高频创作者

特征：

- 经常做图
- 会反复迭代
- 对结果稳定性要求高

核心诉求：

- 想减少重复试错
- 想保存可复用方法

### 3.2 电商 / 设计 / 内容运营

特征：

- 有明确业务场景
- 常做商品图、海报图、封面图、批量素材

核心诉求：

- 想快速复制成功流程
- 想批量探索变体

### 3.3 进阶型个人用户

特征：

- 对 Prompt 和参数感兴趣
- 愿意花时间调结果

核心诉求：

- 想看到系统到底是怎么生成的
- 想自己控制更多细节

### 3.4 BYOK / 多 Provider 用户

特征：

- 关心模型、网关、成本和执行策略

核心诉求：

- 想自己决定走哪个 Provider
- 想在失败时快速切换执行路径

## 4. 专业版核心目标

### 4.1 用户目标

- 看得到系统使用了什么模板和 Prompt
- 控得住更多结果参数
- 能从历史成功结果直接重跑或变体
- 能理解每个结果来自哪一步
- 能把单次出图升级为重复可用的方法

### 4.2 平台目标

- 支撑更多深度能力而不污染普通版
- 把 Prompt、模板、参数、版本链系统化沉淀
- 为模板复用、批量生产、Provider 策略和后续高级权益打底

## 5. 核心设计原则

- 控制力优先于极简
- 透明度优先于完全隐藏
- 复用优先于一次性完成
- 参数要分组，不可平铺
- 高级能力可见，但不可无序
- 历史与版本必须可追溯
- 专业版是工作台，不是调试台

## 6. 专业版与普通版的本质区别

### 6.1 普通版

- 任务导向
- 动作语言
- 少输入少决策
- 系统主动补全
- 默认隐藏专业细节

### 6.2 专业版

- 控制导向
- 能力语言可见
- 参数可调
- Prompt 可见可改
- 模板与版本可复用

### 6.3 结论

普通版是“做图助手”。  
专业版是“创作控制台”。

## 7. 专业版核心能力模块

专业版一期建议至少覆盖以下 8 个模块。

## 7.1 高级创作控制

包括：

- 更完整的尺寸、数量、质量控制
- 参考图控制
- 局部编辑控制
- 风格强度 / 保真倾向
- 多结果策略
- 相似图 / 变体控制

目标：

- 让用户更精确控制结果方向

## 7.2 Prompt 可见与可编辑

包括：

- 查看最终 Prompt
- 复制 Prompt
- 在当前结果基础上编辑 Prompt 再跑
- 查看系统补全前后差异

目标：

- 让用户不仅看到结果，也看得到系统是如何组织生成的

## 7.3 模板与模板变体

包括：

- 选择模板
- 查看模板结构
- 切换模板变体
- 保存为我的常用模板
- 从历史结果反向沉淀模板草稿

目标：

- 让模板成为生产资产，而不是参考文案

## 7.4 任务参数快照

包括：

- 每次任务完整记录参数
- 一键复制参数重跑
- 与另一版本做参数对比
- 参数回退

目标：

- 保证复用与对比能力

## 7.5 历史与版本链

包括：

- 看到每张结果来自哪一步
- 结果之间的父子关系
- 从任意节点继续创作
- 标记最终版 / 候选版

目标：

- 让历史不再只是图片列表

## 7.6 Provider 与模型控制

包括：

- 选择默认 Provider
- 选择 model
- 查看当前走的平台模式还是自带模式
- 失败时切换执行策略

目标：

- 给高级用户更清晰的执行掌控感

## 7.7 批量与生产能力

包括：

- 一次出多张
- 同一模板多变体
- 同一主体多背景
- 同一需求多风格探索

目标：

- 从创作走向生产

## 7.8 资产与导出

包括：

- 下载原图和结果图
- 导出 Prompt
- 导出参数快照
- 保存到项目或作品集

目标：

- 支撑复用、协作和归档

## 8. 专业版页面范围

专业版至少应覆盖以下页面或视图：

- 专业版创作页
- 专业版结果页
- 专业版历史与版本页
- 专业版模板视图
- 专业版 Provider / 执行设置视图

其中一期最关键的是：

- 专业版创作页
- 专业版结果页

## 9. 专业版页面定位

## 9.1 专业版创作页

核心任务：

- 发起任务
- 调整高级参数
- 查看和修改 Prompt
- 选择模板与 Provider

应重点承载：

- 参数控制
- Prompt 区
- 模板选择
- 结果预览

## 9.2 专业版结果页

核心任务：

- 比较结果
- 查看生成上下文
- 基于当前结果继续高精度修改
- 复制参数与 Prompt 重跑

应重点承载：

- 结果对比
- Prompt 与参数快照
- 版本链
- 继续修改入口

## 9.3 专业版历史页

核心任务：

- 快速筛选
- 快速复用
- 快速回到某个版本继续创作

## 9.4 专业版模板视图

核心任务：

- 理解模板结构
- 选择变体
- 保存常用模板

## 10. 专业版信息暴露原则

专业版允许完整暴露这些内容：

- 模板名称与来源
- Prompt 内容
- 结构化参数
- 更多高级设置
- Provider 选择
- 运行模式信息
- 历史参数复制
- 版本关系细节

但仍需遵循以下规则：

- 参数必须分组
- 不能平铺无序
- 需要默认推荐值
- 需要清晰的主次层级

## 11. 专业版与普通版的协同关系

### 11.1 共享内容

- 同一套任务系统
- 同一套历史结果
- 同一套资产数据
- 同一套版本链
- 同一套模板基础设施

### 11.2 区别内容

- 表现层
- 信息暴露层级
- 可操作能力层级

### 11.3 切换要求

- 切换模式不丢上下文
- 不清空当前输入
- 不丢图片与结果
- 不要求重新发起任务

## 12. 一期范围

### 12.1 P0 必做

- 专业版创作页基础版
- 专业版结果页基础版
- Prompt 查看与复制
- 高级参数分组
- 模板查看与选择
- 参数快照记录
- 结果版本父子关系
- Provider / model 基础选择
- 普通版与专业版切换保持上下文

### 12.2 P1 建议做

- Prompt 编辑后重跑
- 参数对比
- 常用模板保存
- 结果版本标记
- 批量生成入口
- 失败后的 Provider 切换

### 12.3 P2 后续做

- 模板草稿沉淀
- 模板 CMS 联动
- 更完整批量生产能力
- 项目级归档
- 高级导出能力

## 13. 一期明确不做

- 完全开放式模板编程
- 企业级协作审批流
- 外部 API 平台
- 复杂自动化工作流可视编排
- 多角色权限系统

## 14. 成功指标

专业版建议关注以下指标：

- 专业版切换率
- Prompt 查看率
- Prompt 复制率
- 参数重跑率
- 历史结果复用率
- 版本链继续创作率
- 模板使用率
- 批量生成使用率

辅助指标：

- Provider 切换使用率
- 单任务平均迭代次数
- 每个结果节点的后续利用率

## 15. 主要风险

### 风险一

专业版只是普通版多几个折叠项。

结果：

- 高级用户仍觉得控制力不足

### 风险二

所有高级能力平铺暴露。

结果：

- 页面变成参数墙
- 可用性下降

### 风险三

Prompt、模板、参数、版本各自孤立。

结果：

- 虽然信息变多，但无法真正复用

### 风险四

专业版与普通版上下文割裂。

结果：

- 切换成本高
- 用户体验断裂

## 16. 文档结论

专业版不是普通版的附属页，而是图像 Skill 平台中承担“控制、复用、生产”的核心工作台。

它必须做到三件事：

- 看得见系统如何工作
- 控得住更细的结果方向
- 用得起过去的成功经验

只要这三点成立，专业版就有独立存在价值，并且不会反过来污染普通用户版体验。
