# 图像 Skill 平台 V1 · 专业版线框级信息结构

## 文档说明

- 文档目标：把专业版核心页面继续拆到线框级信息结构，作为后续原型和界面设计依据。
- 文档定位：承接 [image-skill-platform-pro-pages-v1.md](./image-skill-platform-pro-pages-v1.md) 的线框级文档。
- 当前状态：专业版线框级结构文档。
- 更新时间：2026-05-05。

## 1. 文档范围

本文件优先覆盖专业版中最关键的两张页面：

1. 专业版创作页
2. 专业版结果页

说明：

- 专业版历史与版本页、模板视图、Provider / 执行设置视图暂先作为后续扩展页处理
- 先把最关键主链路的工作台和结果页定清楚

## 2. 专业版线框设计原则

专业版线框结构应遵循以下原则：

- 结果区必须足够大
- 参数区必须分组
- Prompt 区必须可见但不抢主结果
- 高级能力不能平铺无序
- 结果、参数、Prompt、版本之间必须形成协同关系

一句话：

专业版不是“更多内容”，而是“更有层级的控制界面”。

## 3. 专业版创作页线框结构

### 3.1 页面目标

- 让用户高控制地发起生成或编辑任务
- 让用户可看见模板、Prompt、参数和执行方式
- 让结果预览和控制能力同时存在，但互不抢主线

### 3.2 页面整体结构

建议采用三栏布局，但比普通版更强调功能分区：

1. 左侧：模板、Prompt、参数控制区
2. 中间：结果预览与候选结果区
3. 右侧：执行信息、版本链、任务快照区

### 3.3 创作页线框草图说明

### 3.4 顶部条

建议包含：

- 返回
- 当前任务标题
- 保存草稿
- 返回简洁模式
- 当前 Provider / model 状态
- 主执行按钮组

设计要求：

- 顶部条承载全局动作
- 不把所有控制都塞进顶部

### 3.5 左侧控制区

左侧建议按以下顺序组织：

1. 任务类型
2. 模板选择
3. Prompt 区
4. 图片输入区
5. 参数分组区
6. 执行设置区
7. 主操作区

#### 3.5.1 任务类型区

作用：

- 明确当前任务是生成、编辑还是局部编辑

建议项：

- generate
- edit
- inpaint

#### 3.5.2 模板选择区

作用：

- 让用户显式选择模板与变体

建议元素：

- 模板卡
- 模板分类入口
- 模板变体切换
- 当前模板说明

#### 3.5.3 Prompt 区

作用：

- 让专业用户看到并干预 Prompt

建议元素：

- 原始描述输入
- 系统补全结果查看
- 最终 Prompt 预览
- 复制 Prompt
- 编辑后重跑

#### 3.5.4 图片输入区

作用：

- 承载原图、参考图和蒙版

建议元素：

- 上传原图
- 上传参考图
- 上传蒙版
- 图片预览缩略卡

#### 3.5.5 参数分组区

作用：

- 提供更细粒度控制，但必须有分组

建议分组：

- 构图与比例
- 风格与强度
- 输出与质量
- 高级控制

设计要求：

- 不允许一大屏无分组参数平铺

#### 3.5.6 执行设置区

作用：

- 管理 Provider / model / 执行策略

建议元素：

- Provider 选择
- model 选择
- 执行模式状态
- 失败切换策略

#### 3.5.7 主操作区

建议按钮：

- 生成
- 生成 4 张
- 基于当前重跑

### 3.6 中间结果区

中间结果区仍然是视觉中心。

建议承载：

- 主结果大图
- 候选结果条
- 生成过程状态

设计要求：

- 结果比参数更重要
- 即使是专业版，也不能让用户“只看到面板看不到图”

### 3.7 右侧信息区

右侧建议只放“与当前执行相关的上下文信息”，包括：

- 当前任务摘要
- 当前模板
- 当前 Provider / model
- 当前运行模式
- Prompt 快照
- 参数快照
- 版本链
- 任务日志 / 最近一次执行

设计要求：

- 右侧是“看懂这次任务”的区，而不是主操作区

## 4. 专业版结果页线框结构

### 4.1 页面目标

- 让用户分析结果
- 让用户对比候选
- 让用户理解 Prompt、参数和版本关系
- 让用户继续高精度修改

### 4.2 页面整体结构

建议采用三段式结构：

1. 上方：主结果展示区
2. 中段：结果对比与继续修改动作区
3. 下方或右侧：Prompt、参数、版本、执行信息区

### 4.3 结果页线框草图说明

```text
+---------------------------------------------------------------------------------------------------------------+
| 顶部：返回创作页 | 当前结果标题 | 保存到作品 | 复制 Prompt | 复制参数重跑 | 返回简洁模式                      |
+---------------------------------------------------------------------------------------------------------------+
| 主结果展示区                                                                                                 |
| [当前主结果大图]                                                                                             |
+---------------------------------------------------------------------------------------------------------------+
| 候选结果对比区                                                                                               |
| [缩略图1] [缩略图2] [缩略图3] [缩略图4]                                                                      |
+---------------------------------------------------------------------------------------------------------------+
| 继续修改动作区                                                                                               |
| [更像这一张] [换个风格] [只改这一部分] [保留主体换背景] [再生成几张] [基于当前参数重跑]                     |
+---------------------------------------------------------------------------------------------------------------+
| 下方信息区：左侧结果说明 / 中间 Prompt 与参数 / 右侧版本与执行信息                                           |
|---------------------------|--------------------------------------|------------------------------------|
| 当前结果说明               | Prompt 快照                          | 版本链                              |
| 当前风格说明               | 最终 Prompt                          | 父节点 / 当前节点 / 子节点          |
| 系统保护说明               | 参数快照                             |                                    |
|                           | 尺寸 / 质量 / 数量 / 模板 / model     | 执行记录 / 任务日志                 |
+---------------------------------------------------------------------------------------------------------------+
```

### 4.4 顶部条

建议包含：

- 返回创作页
- 当前结果标题
- 保存到作品
- 复制 Prompt
- 复制参数重跑
- 返回简洁模式

设计要求：

- 顶部动作偏向复用和切换
- 下载、收藏等可保留，但不必全部抢顶部主位

### 4.5 主结果展示区

作用：

- 让用户先判断结果质量

设计要求：

- 主结果足够大
- 支持放大查看
- 支持高效切换候选

### 4.6 候选结果对比区

作用：

- 对比多张候选结果

建议元素：

- 候选缩略图列表
- 当前选中高亮
- 差异提示

### 4.7 继续修改动作区

专业版结果页也需要保留高频动作，但会比普通版多一些“复用导向”的动作。

建议动作：

- 更像这一张
- 换个风格
- 只改这一部分
- 保留主体换背景
- 再生成几张
- 基于当前参数重跑

说明：

- 普通版强调“继续改”
- 专业版既强调“继续改”，也强调“重跑与复用”

### 4.8 信息区

专业版结果页的信息区建议分成三组：

#### 左侧：结果说明

- 当前风格说明
- 系统保护说明
- 当前结果定位

#### 中间：Prompt 与参数

- Prompt 快照
- 最终 Prompt
- 参数快照
- 模板信息
- 尺寸 / 质量 / 数量

#### 右侧：版本与执行信息

- 版本链
- 父节点 / 当前节点 / 子节点
- 执行记录
- 任务日志
- Provider / model

## 5. 为什么先只拆这两页

原因：

- 专业版主工作流主要发生在创作页和结果页
- 历史页、模板视图、Provider 视图更多是增强和管理页
- 如果这两页不成立，专业版就不成立

因此：

- 先把专业版主链路的页面结构定稳
- 再继续拆历史页、模板视图和 Provider 视图

## 6. 专业版两页与普通版的关系

### 6.1 创作页关系

普通版创作页：

- 少输入
- 多引导
- 多补全

专业版创作页：

- 可看模板
- 可看 Prompt
- 可调参数
- 可选 Provider

### 6.2 结果页关系

普通版结果页：

- 先看图
- 顺手继续改

专业版结果页：

- 看图
- 看 Prompt
- 看参数
- 看版本
- 看执行信息
- 再决定怎么重跑或继续改

## 7. 文档结论

专业版线框结构的核心不是把更多东西堆到页面里，而是明确：

- 什么应该放左侧控制
- 什么应该放中间看结果
- 什么应该放右侧做上下文解释

只要专业版创作页和结果页的这三层关系清楚，后续组件和交互设计就能继续稳定推进。
