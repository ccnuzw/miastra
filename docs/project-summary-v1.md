# Miastra Studio 项目整理稿 v1

## 项目定位
Miastra Studio 面向个人创作者的 AI 图片创作工作台，当前阶段 MVP 已成型，正进入工作流打磨阶段。

## 当前功能树
### A. 生图能力
- 文生图
- 图生图
- OpenAI Images API 兼容
- 多参考图 / 图生图增强

### B. 创作参数
- Prompt / 负面提示词
- 画幅、分辨率、质量
- 细节强度
- 风格 token
- 流式生成

### C. 批量抽卡
- 并发控制
- 延迟控制
- 超时控制
- 自动重试
- 安全模式
- 变化强度 / 维度
- 快捷预设

### D. 队列控制
- 暂停 / 恢复
- 单任务取消
- 取消全部
- 失败重试
- 状态可视化

### E. Prompt 资产
- Prompt 模板库
- 模板复用
- 最近使用

### F. 作品管理
- 图片墙 / Rail
- 预览
- 收藏
- 标签
- 搜索筛选
- 批量选择
- 删除

### G. 作品复用
- 参数快照
- 回填旧参数
- 再次生成
- 批次信息保留

### H. 导出能力
- 单图下载
- ZIP 下载
- `metadata.json`

### I. 配置与部署
- Provider 本地配置
- 连接测试
- `/sub2api` 代理
- Vite
- Docker / Docker Compose
- Nginx 代理

## 当前版本判断
### 已具备
- 主流程完整
- 批量创作能力明显
- 作品可管理、可复用、可导出
- 基础部署链路已补齐

### 主要短板
- 缺少后端业务层
- 缺少账号体系与协作能力
- 缺少云端同步与对象存储
- 缺少测试与 CI/CD
- 依赖本地持久化，跨设备能力弱

## 下一阶段 Roadmap
### Phase 1：从“能用”到“稳定好用”
- 模板体系增强
- 作品搜索 / 标签 / 详情增强
- 队列与失败项重试优化
- 错误提示标准化
- 基础测试补齐

### Phase 2：从“个人工具”到“正式产品”
- 后端代理与统一错误处理
- 用户体系
- 云端存储与数据库
- 多设备同步
- 导出与资产管理标准化

### Phase 3：从“产品”到“平台”
- 更多 Provider 接入
- 能力探测与参数映射
- 模板 / 资产生态
- 更完善的数据与协作能力

## 推荐优先级 Backlog
| 优先级 | 建议内容 |
|---|---|
| P0 | 模板搜索、作品搜索增强、失败项一键重试、错误提示标准化、ZIP 导出兼容、基础 Smoke Test |
| P1 | 模板分类 / 标签、作品批量标签、批量收藏、批次视图增强、复跑入口 |
| P2 | 模板最近使用排序、作品详情增强、文档与体验细化、更多 Provider 适配 |

## 下一版本建议定义
v0.2 = 从本地生图台升级为稳定创作台。

## 简版 PRD 草案
- 目标：提升模板复用、资产管理与批量生成稳定性。
- 用户：个人创作者、重度试验型图片生成用户。
- 问题：能生成，但不够好找、好管、好复用、好恢复。
- 方案：补齐模板资产、作品管理、队列控制与工程稳定性。
- 成功指标：模板复用率提升、失败恢复效率提升、导出成功率稳定、日常生成流程更顺畅。
