# Miastra Studio v0.2 开发任务拆分清单

## 0. 任务拆分原则
- **P0**：直接影响 v0.2 能否发布，属于主流程、稳定性、导出或回归门槛。
- **P1**：显著提升体验，但在极端情况下可延后。
- **P2**：锦上添花或可裁剪项，资源不足时优先砍掉。

## 1. 版本总目标
把现有本地生图能力整理为一个稳定、可复用、可管理的创作台，补齐模板资产、作品管理、队列恢复与工程稳定性。

## 2. 前端任务清单
| 编号 | 优先级 | 目标 | 涉及模块 | 任务点 | 验收 |
|---|---|---|---|---|---|
| FE-01 模板搜索 | P0 | 让用户快速找到模板 | Prompt 模板库、搜索栏、状态管理 | 支持关键词检索、空态、结果高亮、搜索防抖 | 输入关键词后可准确筛出模板，刷新后行为一致 |
| FE-02 模板分类/标签 | P0 | 让模板可组织、可筛选 | 模板列表、标签组件、筛选面板 | 展示分类、标签筛选、组合过滤、保存元数据 | 分类与标签可正常筛选且结果正确 |
| FE-03 模板编辑复制 | P0 | 避免直接覆盖原模板 | 模板详情、编辑弹窗、复制动作 | 支持复制为新模板、独立编辑、保存后生成新条目 | 复制后原模板不变，新模板可独立使用 |
| FE-04 作品批量标签 | P0 | 降低作品整理成本 | 作品列表、批量工具栏、标签管理 | 多选作品、批量增删标签、批量保存 | 批量操作后标签状态正确持久化 |
| FE-05 作品搜索增强 | P0 | 提升作品查找效率 | 作品库、搜索栏、筛选器 | 支持标题、标签、批次、关键参数等维度检索 | 搜索结果稳定且可组合筛选 |
| FE-06 作品详情增强 | P1 | 让作品信息可追溯 | 作品详情页、侧栏、参数面板 | 展示参数快照、参考图、批次信息、导出信息 | 详情页可完整查看关键元数据 |
| FE-07 队列失败项一键重试 | P0 | 降低失败恢复成本 | 队列列表、失败项操作区 | 单项重试、状态刷新、防重复提交 | 失败任务可单独重试且状态更新正确 |
| FE-08 批次视图增强 | P1 | 让批次状态更直观 | 批次页面、进度条、统计卡片 | 展示成功/失败/进行中数量、批次时间线 | 批次状态在前端可清晰查看 |
| FE-09 批次复跑入口 | P1 | 让历史批次可再次执行 | 批次详情、历史记录入口 | 从历史批次恢复参数并发起新任务 | 可从历史批次一键进入复跑流程 |
| FE-10 错误提示标准化 | P0 | 让错误可读、可处理 | 全局错误处理、Toast、Dialog | 统一错误文案、统一展示样式、统一恢复建议 | 同类错误展示一致，用户可判断处理方式 |
| FE-11 批量收藏 | P1 | 让作品筛选和二次整理更快 | 作品库、收藏状态、批量工具栏 | 多选收藏/取消收藏、批量保存 | 收藏状态持久化且刷新后不丢失 |
| FE-12 模板最近使用排序 | P2 | 让高频模板更易触达 | 模板列表、排序选项、最近使用记录 | 记录使用时间、按最近使用排序 | 最近常用模板可稳定排在前面 |

## 3. 后端任务清单
| 编号 | 优先级 | 目标 | 涉及模块 | 任务点 | 验收 |
|---|---|---|---|---|---|
| BE-01 代理能力预留 | P1 | 为未来后端代理留接口 | 代理层设计、接口契约 | 预留统一代理入口、配置占位、能力扩展点 | 当前不影响前端，未来可平滑接入 |
| BE-02 错误码映射 | P0 | 统一 Provider 错误到 UI 错误 | 错误映射层、响应结构 | 定义错误码、状态码到文案映射 | 同类错误返回同类结构与提示 |
| BE-03 批量任务代理评估 | P2 | 评估批量提交/轮询是否需要后端承担 | 批量任务链路、队列能力 | 评估轮询、重试、幂等与限流 | 输出评估结论与改造边界 |
| BE-04 导出数据结构兼容文档 | P0 | 保证导出结构未来可兼容 | metadata、导出目录、字段说明 | 记录字段含义、兼容规则、升级说明 | 文档可直接指导前后端协作 |

## 4. 测试任务清单
| 编号 | 优先级 | 目标 | 涉及模块 | 任务点 | 验收 |
|---|---|---|---|---|---|
| QA-01 parser 测试 | P0 | 覆盖模板与参数解析 | Prompt parser、模板渲染 | 测试输入、变量、特殊字符、空值 | 解析结果稳定，无明显回归 |
| QA-02 URL / 请求测试 | P0 | 覆盖 Provider 配置与代理请求 | API URL、请求封装、`/sub2api` | 检查请求拼接、超时、header 传递 | 关键请求链路可通过用例验证 |
| QA-03 存储兼容测试 | P0 | 确保本地数据可持续读写 | localStorage、IndexedDB、数据迁移 | 测试旧数据读取、新结构写入 | 升级后历史数据不丢失 |
| QA-04 ZIP 导出测试 | P0 | 验证导出包结构 | 导出流程、metadata.json、压缩包 | 检查目录结构、文件命名、元数据 | ZIP 可正常解压且结构稳定 |
| QA-05 队列行为测试 | P0 | 验证暂停、恢复、取消、重试 | 队列控制、状态流转 | 测试单项取消、整批取消、失败重试 | 队列状态流转符合预期 |
| QA-06 手工 Smoke Test | P0 | 发布前最小人工回归 | 核心主流程 | 从模板、生成、队列、作品、导出跑一遍 | 核心链路可手工稳定完成 |

## 6. 推荐迭代排期
### Sprint 1
- 完成模板搜索、分类、复制。
- 完成作品搜索增强、错误提示标准化。
- 同步建立 parser / URL / 存储的基础测试项。

### Sprint 2
- 完成作品批量标签、批量收藏、作品详情增强。
- 完成队列失败项一键重试、批次视图、批次复跑入口。
- 补齐队列行为测试与 ZIP 导出测试。

### Sprint 3
- 完成模板最近使用排序。
- 完成导出结构兼容文档、发布说明与 Smoke Checklist。
- 集中修复回归并进行发布前验证。

## 8. 建议负责分工
- **前端**：FE-01 ~ FE-12，负责界面、交互、状态管理与导出体验。
- **测试**：QA-01 ~ QA-06，负责回归场景、 Smoke Test 与验收清单。
- **后端运维 / 代理支持**：BE-01 ~ BE-04，负责代理边界、错误映射、导出兼容文档。
- **产品 / 文档**：PM-01 ~ PM-03，负责范围冻结、发布说明、验收标准。

## 9. 最终建议
如果资源有限，建议先做最小闭环 6 项：
1. FE-01 模板搜索
2. FE-03 模板编辑复制
3. FE-05 作品搜索增强
4. FE-07 队列失败项一键重试
5. FE-10 错误提示标准化
6. QA-04 ZIP 导出测试
