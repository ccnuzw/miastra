# Miastra Studio v0.2 任务排期表（按周 / 按 Sprint）

## 1. 排期目标
基于 2026-05-01 的当前进度，在 6 周内完成 v0.2 的核心增强、稳定性补齐与发布验收，把当前本地生图台升级为稳定、可复用、可管理的创作台。

## 2. 总体节奏概览
- **Sprint 1**：2026-05-04 ~ 2026-05-15
- **Sprint 2**：2026-05-18 ~ 2026-05-29
- **Sprint 3**：2026-06-01 ~ 2026-06-12

## 4. 按 Sprint 排期表
### Sprint 1
- **Sprint Goal**：让模板和作品更好找、更好改。
- **范围**：模板搜索、分类/标签、复制；作品搜索增强；错误提示标准化。
- **交付结果**：模板库具备基础组织能力，常见错误提示可读。
- **出口标准**：模板和搜索主路径稳定可用，构建通过。

### Sprint 2
- **Sprint Goal**：让批量任务更可控、可恢复。
- **范围**：批量标签、批量收藏、作品详情增强；失败项重试、批次视图、复跑入口。
- **交付结果**：生成结果可以按资产维度整理，失败任务可快速恢复。
- **出口标准**：批量流程与队列状态一致，关键操作无阻塞。

### Sprint 3
- **Sprint Goal**：完成稳定化、导出和发布准备。
- **范围**：模板最近使用排序；ZIP 导出兼容；数据兼容检查；Smoke Test；发布文档。
- **交付结果**：v0.2 RC 可供正式发布评估。
- **出口标准**：最小发布清单通过，核心流程可重复验证。

## 5. 角色分工建议
- **前端**：主导模板、作品、队列、导出与交互体验。
- **测试**：主导 Smoke Test、回归测试、导出与兼容性验证。
- **后端运维**：主导代理兼容、错误映射、部署验证与发布检查。

## 6. 每周例会建议
- **周一**：确认本周目标与优先级。
- **周三**：同步风险、阻塞与接口问题。
- **周五**：进行阶段验收、回顾和下周预判。

## 7. 风险与缓冲
- Provider 兼容性差异可能影响请求与错误提示一致性。
- 本地存储结构变化可能引入历史数据兼容风险。
- 批量队列与导出功能容易出现边界回归。
- 建议在 Week 5 ~ Week 6 预留至少 1 个缓冲窗口用于修复。

## 8. 建议的裁剪顺序
1. 模板最近使用排序
2. 作品详情中的非关键展示项
3. 批量收藏
4. 批次复跑入口
5. 更细的分类体系
6. 非核心 Provider 扩展

## 9. 最小可发布版本（v0.2 Minimum Ship）
- 模板搜索 / 复制可用
- 作品搜索增强可用
- 失败项一键重试可用
- 错误提示标准化可用
- ZIP 导出与 metadata 可用
- Smoke Test 通过

## 10. 最终发布时间建议
- **优先建议**：2026-06-12
- **保守建议**：2026-06-15
